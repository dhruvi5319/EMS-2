# Plan 05-01: Evidence API Backend

**Phase:** 5 — Evidence, Findings, and Gate P3  
**Status:** not started  
**Features:** F8  

## Goal

Implement the evidence item CRUD backend with multer file upload, StorageProvider integration, and sensitivity-based access control.

## Tasks

### Task 1 — Evidence repository and service
- Create `src/modules/evidence/evidence.repository.ts`:
  - listEvidence(engagementId, userRoles) — excludes restricted items for AL/RO
  - getEvidence(id, userRoles) — same sensitivity guard
  - createEvidence(data)
  - updateEvidence(id, data)
  - softDeleteEvidence(id) — blocked if linked to objectives/findings/statements
  - addEvidenceFile(evidenceId, fileData)
  - getEvidenceFiles(evidenceId)
  - getEvidenceFile(evidenceId, fileId) — for download
- Create `src/modules/evidence/evidence.service.ts`:
  - PRIVILEGED_ROLES: ['AN', 'EM', 'QA', 'IR', 'PC', 'AD'] (AL and RO excluded from restricted)
  - canViewRestricted(roles) helper
  - All CRUD with sensitivity enforcement

### Task 2 — Evidence routes
- Create `src/modules/evidence/evidence.controller.ts` mounted at `/api/engagements/:id/evidence`:
  - `GET /api/engagements/:id/evidence` — list evidence items
  - `POST /api/engagements/:id/evidence` — create evidence item (AN, EM, AD)
  - `GET /api/engagements/:id/evidence/:evidenceId` — get single evidence item
  - `PATCH /api/engagements/:id/evidence/:evidenceId` — update (AN, EM, AD)
  - `DELETE /api/engagements/:id/evidence/:evidenceId` — soft delete (AN, EM, AD)
  - `POST /api/engagements/:id/evidence/:evidenceId/files` — upload file (multer memory; ≤25 MB)
  - `GET /api/engagements/:id/evidence/:evidenceId/files` — list files with filename
  - `GET /api/engagements/:id/evidence/:evidenceId/files/:fileId` — stream file download

## Acceptance Checks
- [ ] Create and list evidence items works
- [ ] AL/RO cannot see restricted evidence items (returns filtered list)
- [ ] File upload stores file via StorageProvider; returns file metadata
- [ ] Re-fetching after upload shows correct filenames
- [ ] Soft delete blocked if evidence is linked to objectives
- [ ] File download streams correctly; access-controlled
