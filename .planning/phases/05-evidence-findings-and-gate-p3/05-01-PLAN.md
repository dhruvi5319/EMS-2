
**Types:**
```typescript
export type EvidenceType = 'document' | 'dataset' | 'interview_note' | 'meeting_note' | 'other';
export type Sensitivity = 'standard' | 'restricted';

export interface EvidenceItem {
  id: string;
  engagement_id: string;
  evidence_type: EvidenceType;
  source: string;
  date_received: string;
  custodian: string | null;
  description: string | null;
  sensitivity: Sensitivity;
  created_by: string;
  created_at: string;
  updated_at: string;
  file_count?: number;
  objective_count?: number;
}

export interface EvidenceFile {
  id: string;
  evidence_item_id: string;
  original_filename: string;
  file_size: number;
  mime_type: string;
  storage_key: string;
  uploaded_by: string;
  created_at: string;
}
```

```typescript
const ALLOWED_MIME_TYPES = new Set([
```

**`listEvidence(engagementId, filters, viewerRoles)`:**
- `filters`: `{ type?: EvidenceType, sensitivity?: Sensitivity, limit?: number, offset?: number }`
- `viewerRoles`: string[] from req.user.roles
- If viewer roles include ONLY 'AL' or 'RO' (i.e., does NOT include any of AN/EM/QA/IR/PC/AD):
  - Add `WHERE sensitivity = 'standard'` to filter out restricted items entirely
- SELECT evidence_items with:
  - `(SELECT COUNT(*) FROM evidence_files WHERE evidence_item_id = e.id)` AS file_count
  - `(SELECT COUNT(*) FROM evidence_objective_links WHERE evidence_item_id = e.id)` AS objective_count
- ORDER BY `date_received DESC, created_at DESC`
- Apply limit (default 20, max 100) and offset
- Returns `{ evidence: EvidenceItem[], total: number }`

Helper to determine if viewer is restricted-only:
```typescript
const RESTRICTED_EXCLUDED_ROLES = new Set(['AL', 'RO']);
const PRIVILEGED_ROLES = new Set(['AN', 'EM', 'QA', 'IR', 'PC', 'AD']);
function canViewRestricted(roles: string[]): boolean {
  return roles.some(r => PRIVILEGED_ROLES.has(r));
}
```

**`createEvidence(engagementId, data, actorId)`:**
- `data`: `{ evidence_type, source, date_received, custodian?, description?, sensitivity }`
- Validate:
  - `evidence_type` must be one of: document/dataset/interview_note/meeting_note/other — throw `{ status: 422, message: 'Invalid evidence type.' }` if not
  - `source` non-empty, ≤500 chars
  - `date_received` is a valid date string (ISO or MM/DD/YYYY — parse with `new Date()`; throw 422 if invalid)
  - `sensitivity` must be 'standard' or 'restricted' — throw 422 if not
- INSERT into `evidence_items` with UUID id, set `created_by = actorId`
- Return created `EvidenceItem`

**`updateEvidence(engagementId, evidenceId, data, actorId)`:**
- Verify evidence item exists and belongs to this engagement (throw 404 if not)
- Validate same rules as createEvidence for any fields provided (partial update)
- UPDATE `evidence_items` with provided fields + `updated_at = NOW()`
- Return updated `EvidenceItem`

**`deleteEvidence(engagementId, evidenceId)`:**
- Verify evidence item exists and belongs to this engagement (throw 404 if not)
- Check: `SELECT COUNT(*) FROM evidence_objective_links WHERE evidence_item_id = :id`
- Check: `SELECT COUNT(*) FROM finding_evidence_links WHERE evidence_item_id = :id`
- If either count > 0: throw `{ status: 409, message: 'Cannot delete — this evidence item is linked to objectives or findings. Unlink it first.' }`
- In a DB transaction:
  1. Delete all files from storage (use StorageProvider) — query `evidence_files` for this item
  2. DELETE from `evidence_files` WHERE `evidence_item_id = :id`
  3. DELETE from `evidence_items` WHERE `id = :id`
- Return `{ message: 'Evidence item deleted.' }`

**`uploadFile(engagementId, evidenceId, file, actorId)`:**
- `file` shape from multer: `{ originalname, mimetype, size, buffer }`
- Verify evidence item exists and belongs to engagement (throw 404 if not)
- Validate MIME type against ALLOWED_MIME_TYPES — throw `{ status: 422, message: 'File type not permitted.' }` if not
- Validate file size ≤ MAX_FILE_SIZE_BYTES — throw `{ status: 422, message: 'File exceeds maximum size of 50 MB.' }` if over
- Count existing files: `SELECT COUNT(*) FROM evidence_files WHERE evidence_item_id = :id`
- If count >= 20: throw `{ status: 422, message: 'Maximum of 20 files per evidence item.' }`
- Use StorageProvider to save file: import from `'../services/storage.service'` (pattern from Phase 3 — `StorageProvider.save(buffer, filename)` returns `storage_key`)
  - storage_key format: `evidence/${evidenceId}/${Date.now()}-${file.originalname}`
- INSERT into `evidence_files`: `{ id: uuid, evidence_item_id: evidenceId, original_filename: file.originalname, file_size: file.size, mime_type: file.mimetype, storage_key, uploaded_by: actorId, created_at: NOW() }`
- Return `EvidenceFile`

**`deleteFile(engagementId, evidenceId, fileId, viewerRoles)`:**
- Verify file exists and belongs to the evidence item in this engagement (JOIN check)
- Use StorageProvider to delete the file by storage_key
- DELETE from `evidence_files` WHERE `id = :fileId`
- Return `{ message: 'File deleted.' }`

**`getEvidenceFile(evidenceId, fileId, viewerRoles)`:**
- Used for download — returns the file for streaming
- Verify file exists (throw 404 if not)
- If the evidence item's sensitivity = 'restricted' AND viewer cannot view restricted: throw `{ status: 403, message: 'Access denied — restricted evidence.' }`
- Return `{ storage_key, original_filename, mime_type }` for the route to stream

Use `db` from `'../db'`. Follow the patterns from `backend/src/services/requests.service.ts` for error shapes.

**CSV export implementation (inline in route, no service function needed):**
```typescript
// GET /evidence/export — returns CSV stream
evidenceRouter.get('/export', requireRole('AL','EM','AN','QA','PC','RO','AD'), async (req, res) => {
  // Excluded: IR role — check explicitly
  if (req.user!.roles.includes('IR') && !req.user!.roles.some(r => ['AL','EM','AN','QA','PC','RO','AD'].includes(r))) {
    // Pure IR user has no other roles that grant access — return 403
    return res.status(403).json({ error: 'Access denied.' });
  }
  const evidence = await listEvidence(req.params.id, { limit: 1000 }, req.user!.roles);
  const rows = evidence.evidence;
  // CSV headers
  const headers = ['ID','Type','Source','Date Received','Custodian','Description','Sensitivity','Objectives Linked','Files'];
  const csvLines = [
    headers.join(','),
    ...rows.map(e => [
      e.id,
      e.evidence_type,
      `"${(e.source || '').replace(/"/g, '""')}"`,
      e.date_received,
      `"${(e.custodian || '').replace(/"/g, '""')}"`,
      `"${(e.description || '').replace(/"/g, '""')}"`,
      e.sensitivity,
      e.objective_count ?? 0,
      e.file_count ?? 0,
    ].join(','))
  ];
  res.setHeader('Content-Type', 'text/csv');
  res.setHeader('Content-Disposition', 'attachment; filename="evidence-registry.csv"');
  res.send(csvLines.join('\n'));
});
```
