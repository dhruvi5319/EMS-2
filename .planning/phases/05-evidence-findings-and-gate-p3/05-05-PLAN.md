# Plan 05-05: Evidence Detail + Gap View UI

**Phase:** 5 — Evidence, Findings, and Gate P3  
**Status:** not started  
**Features:** F8, F9  

## Goal

Build the Evidence Detail page with objective linking popover and the Gap View panel showing objectives with no linked evidence.

## Tasks

### Task 1 — Evidence detail page
- Create `src/hooks/useEvidenceDetail.ts`: fetches single evidence item + files + linked objectives
- Create `src/pages/EvidenceDetailPage.tsx`:
  - Header: evidence type, source, date received, sensitivity badge
  - Files section: list of uploaded files with download links
  - Linked Objectives section: shows objectives this evidence supports
  - "Link to Objective" button → opens LinkObjectivePopover
  - On link success: uses fetchEvidence() re-fetch (not optimistic update — avoids race condition when coverage is null/stale)

### Task 2 — Link objective popover
- Create `src/components/evidence/LinkObjectivePopover.tsx`:
  - Popover with Command (cmdk) listing available objectives (those not yet linked)
  - onMouseDown={(e) => e.preventDefault()} on CommandItem (canonical cmdk v1 + Radix Popover fix)
  - On select: calls `POST /api/engagements/:id/evidence/:evidenceId/objectives`

### Task 3 — Gap view panel
- Create `src/components/evidence/SufficiencyChip.tsx`: color chip for evidence_needed/in_review/sufficient
- Create `src/components/evidence/GapObjectiveCard.tsx`:
  - Shows objective text, information need, evidence count, sufficiency chip
  - Label: "P3 Blocker" (not "Blocker") when evidence_needed with zero evidence
  - onLinkClick callback (not embedded popover — parent context determines link target)
- Create `src/components/evidence/GapViewPanel.tsx`:
  - Lists objectives with zero linked evidence as GapObjectiveCards
  - "No gaps" state when all objectives have evidence

### Task 4 — Wire Gap View into Evidence List page
