# Plan 05-02: Evidence-to-Objective Links + Coverage + Sufficiency API

**Phase:** 5 — Evidence, Findings, and Gate P3  
**Status:** not started  
**Features:** F9  

## Goal

Implement the evidence-to-objective linking, coverage summary (gap view), and objective sufficiency status management APIs.

## Tasks

### Task 1 — Objective-evidence link API
- Create `src/modules/evidence/objectivecoverage.service.ts`:
  - `linkEvidenceToObjective(objectiveId, evidenceItemId, actorId)` — inserts into objective_evidence_links; requires actorId for linked_by NOT NULL
  - `unlinkEvidenceFromObjective(objectiveId, evidenceItemId)` — deletes link
  - `getLinkedEvidence(objectiveId)` — returns evidence items linked to objective
  - `getCoverageSummary(engagementId)` — for each objective: count of linked evidence, list of evidence items
  - `getGapData(engagementId)` — returns objectives with zero linked evidence

- Routes (mounted in evidenceRouter to avoid routing conflicts):
  - `POST /api/engagements/:id/evidence/:evidenceId/objectives` — link evidence to objectives (array of objective_ids)
  - `DELETE /api/engagements/:id/evidence/:evidenceId/objectives/:objectiveId` — unlink
  - `GET /api/engagements/:id/evidence/coverage` — coverage summary per objective

### Task 2 — Objective sufficiency management API
- Add to planning routes:
  - `PATCH /api/engagements/:id/planning/objectives/:objId/sufficiency` — update objectives.status (QA only): evidence_needed / in_review / sufficient
  - `GET /api/engagements/:id/planning/objectives/sufficiency` — list objectives with sufficiency status and evidence count

### Task 3 — CSV export
- Add to evidence routes:
  - `GET /api/engagements/:id/evidence?format=csv` — export evidence registry to CSV (all fields; sensitivity filter applied)

## Acceptance Checks
