**`SufficiencyStatusSelect.tsx`:**
Per UI-SPEC Screen 7 "Objective Sufficiency Table — Status dropdown":
- Uses shadcn `Select` component
- Options: "Evidence Needed" / "In Review" / "Sufficient"
- Current value rendered as trigger showing the current `SufficiencyChip` (from Plan 05-05)
- "In Review" and "Sufficient" options disabled (but visible) if `evidenceCount = 0`:
  - `aria-disabled="true"` on the SelectItem (not HTML `disabled`)
  - Tooltip (shadcn Tooltip) on those items: "Requires at least 1 linked evidence item"
  - `aria-label="Set evidence sufficiency for Objective {N}"` on Select trigger
- On change: calls `PUT /api/engagements/:id/objectives/sufficiency` with `[{ objective_id, sufficiency }]`
  - Loading: spinner in trigger while saving
  - On success: toast "Objective status updated."
  - On error (422): toast error with message
- Access: QA, EM, AD only — if viewer does NOT have any of these roles, render read-only `SufficiencyChip` instead of Select

```typescript
type Props = {
  engagementId: string;
  objectiveId: string;
  objectiveIndex: number;  // for aria-label ("Objective N")
  evidenceCount: number;
  currentStatus: SufficiencyStatus;
  canEdit: boolean;  // true if QA/EM/AD role
  onStatusChanged: (newStatus: SufficiencyStatus) => void;
};
```
