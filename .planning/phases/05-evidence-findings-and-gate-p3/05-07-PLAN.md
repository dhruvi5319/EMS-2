# Plan 05-07: Gate P3 Review UI

**Phase:** 5 — Evidence, Findings, and Gate P3  
**Status:** not started  
**Features:** F10  

## Goal

Build the Gate P3 Review page for QA Reviewers to mark objective sufficiency and approve or return Gate P3.

## Tasks

### Task 1 — Gate P3 review page
- Create `src/pages/GateP3ReviewPage.tsx` (route: `/engagements/:id/gate/p3`):
  - ObjectiveSufficiencyTable: all objectives with SufficiencyStatusSelect (QA can change)
  - P3DecisionPanel (Task 2)
  - "Already Approved" banner: shown when P3 gate_decision status=approved; prevents re-approval

### Task 2 — P3 decision panel
- Create `src/components/gates/P3DecisionPanel.tsx`:
  - Shows P3 prerequisites checklist (reuses P3PrerequisitesChecklist)
  - Textarea: rationale (required)
  - "Approve P3" button:
    - Uses aria-disabled + Tooltip wrapper (not HTML disabled) — allows tooltip on disabled state
    - Disabled when any prerequisite not met
    - AlertDialog confirmation → calls `POST /api/engagements/:id/gate/p3/approve`
  - "Return" button → calls return endpoint (if implemented)
  - On approve: in-place React state update; post-P3-approval green banner uses react-router location state (p3Approved: true) read in EngagementShellPage on mount

### Task 3 — Sufficiency status select
- Create `src/components/findings/SufficiencyStatusSelect.tsx`:
  - shadcn Select with options: evidence_needed / in_review / sufficient
  - QA-only; other roles see read-only chip
  - Calls `PATCH /api/engagements/:id/planning/objectives/:objId/sufficiency`

## Acceptance Checks
- [ ] P3 review page shows all objectives with sufficiency select for QA
- [ ] Non-QA sees read-only sufficiency chips
- [ ] Approve button disabled when any prerequisite fails; tooltip explains why
- [ ] Approve P3 sets engagement.phase to draft; green success banner shown
- [ ] Already-approved banner prevents second approval
- [ ] GateStatusCard P3 updates to approved (emerald) after approval
