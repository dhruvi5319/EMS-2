From TechArch F10 API spec:
```
// FINDINGS (F10)
GET    /api/engagements/:id/findings              // List findings
POST   /api/engagements/:id/findings              // Create finding
// Body: { finding_text, evidence_ids: string[] }
// Response 201: { finding: Finding }

PATCH  /api/engagements/:id/findings/:finding_id  // Update finding text/evidence links/status
DELETE /api/engagements/:id/findings/:finding_id  // Delete finding

// GATE P3 (F10)
GET    /api/engagements/:id/gate/p3/prerequisites  // Check P3 prerequisites — returns blockers[]
POST   /api/engagements/:id/gate/p3               // Record P3 decision
// Body: { decision: 'approved'|'returned', comment?: string }
// On approved: creates GateDecision, AuditEvent (GATE_P3_APPROVED), sets engagement.phase = 'draft'
// On returned: creates GateDecision, AuditEvent (GATE_P3_RETURNED)
// P3 blocked if: any objective has 0 linked evidence OR any objective status = 'evidence_needed'
```

**Types:**
```typescript
export type FindingStatus = 'draft' | 'under_review' | 'accepted' | 'rejected';

export interface Finding {
  id: string;
  engagement_id: string;
  finding_text: string;
  status: FindingStatus;
  created_by: string;
  created_at: string;
  updated_at: string;
  evidence_ids?: string[];
  evidence_count?: number;
}
```

**`listFindings(engagementId, viewerRoles)`:**
- SELECT from `findings` WHERE `engagement_id = :id` ORDER BY `created_at DESC`
- For each finding, attach linked evidence_ids:
  ```sql
  SELECT evidence_item_id FROM finding_evidence_links WHERE finding_id = :finding_id
  ```
  - Alternatively: LEFT JOIN or subquery for efficiency if many findings expected
- Apply sensitivity-based evidence filtering: if viewer cannot view restricted (AL/RO), exclude restricted evidence_ids from the evidence_ids array per finding (they still see the finding, just without restricted evidence items listed)
- Attach `evidence_count` (count of linked evidence items)
- Return `{ findings: Finding[], total: number }`

**`createFinding(engagementId, { finding_text, evidence_ids }, actorId)`:**
- Validate: `finding_text` non-empty — throw `{ status: 422, message: 'Finding text is required.' }`
- INSERT into `findings`: `{ id: uuid, engagement_id, finding_text, status: 'draft', created_by: actorId, created_at: NOW(), updated_at: NOW() }`
- If `evidence_ids` provided (non-empty array):
  - Validate each evidenceId belongs to this engagement: `SELECT id FROM evidence_items WHERE id IN (:ids) AND engagement_id = :engId`
  - INSERT into `finding_evidence_links`: `{ id: uuid, finding_id, evidence_item_id, created_at: NOW() }` for each
- Return new `Finding` with `evidence_ids` and `evidence_count` attached

**`updateFinding(engagementId, findingId, { finding_text?, evidence_ids?, status? }, actorId)`:**
- Verify finding exists and belongs to engagement — throw 404 if not
- If `finding_text` provided: validate non-empty
- If `status` provided: validate is one of 'draft'/'under_review'/'accepted'/'rejected'
- UPDATE `findings` SET relevant fields + `updated_at = NOW()`
- If `evidence_ids` provided (replace strategy):
  - DELETE FROM `finding_evidence_links` WHERE `finding_id = :findingId`
  - INSERT new links for each evidenceId in `evidence_ids`
- Return updated `Finding` with `evidence_ids` and `evidence_count`

**`deleteFinding(engagementId, findingId)`:**
- Verify finding exists and belongs to engagement — throw 404 if not
- In a DB transaction:
  1. DELETE FROM `finding_evidence_links` WHERE `finding_id = :findingId`
  2. DELETE FROM `findings` WHERE `id = :findingId`
- Return `{ message: 'Finding deleted.' }`

**`checkP3Prerequisites(engagementId)`:**
Returns `{ all_pass: boolean, blockers: Array<{ type: string, message: string }> }`

Checks in order:
1. **P2 approved**: `SELECT id FROM gate_decisions WHERE engagement_id = :id AND gate_name = 'P2' AND decision = 'approved' LIMIT 1`
   - If no row: add `{ type: 'p2_not_approved', message: 'Gate P2 has not been approved.' }`

2. **All objectives have ≥1 linked evidence**: import `getObjectiveCoverage` from `'./objectivecoverage.service'`
   - For each objective where `evidence_count = 0`: add `{ type: 'objective_no_evidence', message: 'Objective "{text truncated to 60 chars}" has no linked evidence.' }`

3. **No objective has sufficiency_status = 'evidence_needed'**: from the same coverage result
   - For each objective where `sufficiency_status = 'evidence_needed'`: add `{ type: 'objective_evidence_needed', message: 'Objective "{text truncated to 60 chars}" is marked Evidence Needed.' }`

4. **All findings have ≥1 linked evidence**: 
   - Query: `SELECT f.id, f.finding_text, COUNT(fel.id) AS evidence_count FROM findings f LEFT JOIN finding_evidence_links fel ON fel.finding_id = f.id WHERE f.engagement_id = :id GROUP BY f.id, f.finding_text HAVING COUNT(fel.id) = 0`
   - For each finding with 0 evidence: add `{ type: 'finding_no_evidence', message: 'Finding FD-{N} has no linked evidence.' }`

`all_pass = blockers.length === 0`

**`recordP3Decision(engagementId, { decision, comment }, actorId)`:**
- Validate: `decision` must be 'approved' or 'returned' — throw 422 if not
- Validate: `comment` is required and ≥10 chars — throw `{ status: 422, message: 'Decision comment must be at least 10 characters.' }`
- Re-run `checkP3Prerequisites(engagementId)` — throw `{ status: 422, message: 'P3 prerequisites not met.', blockers }` if `all_pass = false`
- Execute in a **`db.transaction()`**:
  1. INSERT into `gate_decisions`: `{ engagement_id, gate_name: 'P3', decision, rationale: comment, decided_by: actorId, decided_at: NOW() }`
  2. INSERT into `audit_events`: `{ engagement_id, action_code: decision === 'approved' ? 'GATE_P3_APPROVED' : 'GATE_P3_RETURNED', actor_id: actorId, metadata: JSON.stringify({ comment }) }`
  3. If `decision === 'approved'`:
     - UPDATE `engagements SET phase = 'draft', updated_at = NOW()` WHERE `id = :engagementId`
- Return `{ gate_decision: GateDecision }`

Import `getObjectiveCoverage` from `'./objectivecoverage.service'`.
Use `db` from `'../db'`. Follow error shape `{ status, message }` for all throws.

In `backend/src/routes/engagements.ts`, add Gate P3 routes after mounting sub-routers:
```typescript
import { checkP3Prerequisites, recordP3Decision } from '../services/findings.service';

// GET /api/engagements/:id/gate/p3/prerequisites — all roles
engagementsRouter.get('/:id/gate/p3/prerequisites', authenticateSession, async (req, res) => {
  try {
    const result = await checkP3Prerequisites(req.params.id);
    res.json(result);
  } catch (err: any) {
    if (err.status && err.status < 500) return res.status(err.status).json({ error: err.message });
    console.error('P3 prerequisites error:', err);
    res.status(500).json({ error: 'Internal server error' });
  }
});

// POST /api/engagements/:id/gate/p3 — QA, AD only
engagementsRouter.post('/:id/gate/p3', authenticateSession, requireRole('QA', 'AD'), async (req, res) => {
  try {
    const { decision, comment } = req.body;
    const result = await recordP3Decision(req.params.id, { decision, comment }, req.user!.id);
    res.json(result);
  } catch (err: any) {
    if (err.status && err.status < 500) {
      return res.status(err.status).json({ error: err.message, blockers: (err as any).blockers });
    }
    console.error('Gate P3 decision error:', err);
    res.status(500).json({ error: 'Internal server error' });
  }
});
```
