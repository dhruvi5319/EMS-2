# Plan 05-06: Findings List UI

**Phase:** 5 — Evidence, Findings, and Gate P3  
**Status:** not started  
**Features:** F10  

## Goal

Build the Findings list page with finding cards, add finding dialog, objective sufficiency summary, and P3 prerequisites checklist.

## Tasks

### Task 1 — Findings list page
- Create `src/hooks/useFindings.ts`: fetches `GET /api/engagements/:id/findings`
- Create `src/components/findings/FindingCard.tsx`:
  - Shows: finding text, status badge, linked evidence count
  - Edit button (inline text edit or dialog)
  - Delete button (with confirm dialog; disabled if referenced by statements)
- Create `src/pages/FindingsListPage.tsx`:
  - List of FindingCards
  - "Add Finding" button (opens AddFindingDialog)
  - ObjectiveSufficiencySummary component
  - P3ReadinessChecklist component
  - Gate P3 Review link (shown when all prerequisites pass and P3 not yet approved)
  - p3Approved state: disables add/edit when P3 is already approved

### Task 2 — Add finding dialog
- Create `src/components/findings/AddFindingDialog.tsx`:
  - shadcn Dialog
  - Textarea: finding_text (required)
  - Multi-select evidence items to link (at least 1 required)
  - Calls `POST /api/engagements/:id/findings`

### Task 3 — Objective sufficiency summary
- Create `src/components/findings/ObjectiveSufficiencySummary.tsx`:
  - Fetches `GET /api/engagements/:id/planning/objectives/sufficiency`
  - Shows each objective with sufficiency chip (SufficiencyChip)
  - QA can update sufficiency status inline (PATCH sufficiency endpoint)
  - allPass logic: evidence_count > 0 AND sufficiency_status !== 'evidence_needed' (not stricter 'sufficient' check)

### Task 4 — P3 readiness checklist (reuse P2 pattern)
- Create `src/components/findings/P3PrerequisitesChecklist.tsx`:
  - Fetches `GET /api/engagements/:id/gate/p3/prerequisites`
  - Shows all checks with pass/fail icons

## Acceptance Checks
- [ ] Findings list shows all findings with status and evidence count
- [ ] Add finding requires ≥1 evidence link
- [ ] Objective sufficiency chips update when QA changes status
- [ ] allPass is true only when all objectives have evidence AND none are evidence_needed
- [ ] Gate P3 Review link appears when all prerequisites pass
- [ ] p3Approved state prevents editing when P3 is done
