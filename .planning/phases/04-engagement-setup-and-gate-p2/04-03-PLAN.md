# Plan 04-03: Planning Record + Gate P2 Backend

**Phase:** 4 — Engagement Setup and Gate P2  
**Status:** not started  
**Features:** F6, F7  

## Goal

Implement the planning record CRUD (objectives, independence status, design/risk/data-reliability notes), Gate P2 decision with server-side re-validation, and the revision workflow for post-approval edits.

## Tasks

### Task 1 — Planning record and objectives API
- Create `src/modules/planning/planning.repository.ts`, `planning.service.ts`, `planning.controller.ts`:
  - `GET /api/engagements/:id/planning` — get or create planning record; includes objectives
  - `PATCH /api/engagements/:id/planning` — update fields (design_approach, risk_notes, data_reliability_notes, independence_status, schedule_notes); if status=approved, requires revision_note; reverts status to draft
  - `POST /api/engagements/:id/planning/submit` — sets status=ready_for_review; validates non-empty required fields
  - `POST /api/engagements/:id/planning/objectives` — add objective (objective_text, information_need)
  - `PATCH /api/engagements/:id/planning/objectives/:objId` — update objective
  - `DELETE /api/engagements/:id/planning/objectives/:objId` — soft delete (blocked if evidence linked)
  - Writes AuditEvent on submit

### Task 2 — Gate P2 decision API
- Add to `gates.service.ts`:
  - `approveP2(engagementId, actorId, { rationale })`:
    - Server-side re-validates all 8 P2 prerequisites; returns 422 if any fail
    - Sets planning_record.status = approved, approved_by, approved_at
    - Creates GateDecision (gate_type=P2, status=approved)
    - Creates AuditEvent (GATE_P2_APPROVED)
    - All in transaction
  - `returnP2(engagementId, actorId, { rationale })`:
    - Sets planning_record.status = returned
    - Creates GateDecision (gate_type=P2, status=returned)
    - Creates AuditEvent (GATE_P2_RETURNED)
- Routes (QA, AD):
  - `POST /api/engagements/:id/gate/p2/approve`
  - `POST /api/engagements/:id/gate/p2/return`

## Acceptance Checks
- [ ] Planning record created on first GET; updated via PATCH
- [ ] PATCH with approved record requires revision_note; reverts to draft
- [ ] Submit validates design_approach, risk_notes, data_reliability_notes are non-empty
- [ ] P2 approve fails if any prerequisite is missing (server-side check)
- [ ] P2 approve locks planning record (status=approved)
- [ ] P2 return sets status=returned
- [ ] All P2 decisions create GateDecision and AuditEvent
