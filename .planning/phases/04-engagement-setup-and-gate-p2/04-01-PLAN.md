# Plan 04-01: Engagement Backend API

**Phase:** 4 — Engagement Setup and Gate P2  
**Status:** not started  
**Features:** F4  

## Goal

Implement the engagement list, engagement detail (with gates and blockers), and metadata patch APIs.

## Tasks

### Task 1 — Engagement repository and service
- Create `src/modules/engagements/engagements.repository.ts`:
  - listEngagements(userId, userRoles, filters) — scoped by team_assignments for non-AD
  - getEngagement(id) — joins owner, gate_decisions, team_assignments
  - updateEngagement(id, data)
  - getBlockers(engagementId) — computes open blockers
- Create `src/modules/engagements/engagements.service.ts`:
  - `listEngagements(userId, userRoles, { phase, status, owner_id, risk_level })` — with CSV export option
  - `getEngagementDetail(id)` — returns engagement + gate status summary (A1/P2/P3/P4 latest decisions) + blockers
  - `updateEngagement(id, data, actorId)` — validates EM/AD role; writes AuditEvent on changes
  - `getBlockers(id)` — returns array of Blocker objects

### Task 2 — Engagement routes
- Create `src/modules/engagements/engagements.controller.ts`:
  - `GET /api/engagements` — list with filters (EM, QA, AN, IR, PC, AD, AL)
  - `GET /api/engagements/:id` — detail with gates + blockers (all authenticated)
  - `PATCH /api/engagements/:id` — update metadata (EM, AD)
  - `GET /api/engagements/export` — CSV export (registered BEFORE /:id to prevent route capture)

## Acceptance Checks
- [ ] `GET /api/engagements` returns scoped list with gate status summary
- [ ] `GET /api/engagements/:id` returns engagement with A1/P2/P3/P4 gate status cards
- [ ] `PATCH /api/engagements/:id` updates metadata; non-EM/AD returns 403
- [ ] Blockers are computed and returned in the detail response
- [ ] CSV export returns correct Content-Disposition header
