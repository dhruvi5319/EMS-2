# Plan 04-05: Team + Milestones UI

**Phase:** 4 — Engagement Setup and Gate P2  
**Status:** not started  
**Features:** F5  

## Goal

Build the Team and Milestones tab in the Engagement Shell — team member table with add/remove, milestone date inputs with computed status chips.

## Tasks

### Task 1 — shadcn components for team/milestones
- Manually create `src/components/ui/command.tsx` if not yet created (cmdk wrapper)

### Task 2 — Team panel components
- Create `src/hooks/useTeam.ts`: fetches `GET /api/engagements/:id/team`
- Create `src/components/team/TeamMemberTable.tsx`: table of current team with columns: name, role, assigned date, remove button
  - Remove button disabled when: P2 approved, or last EM
  - canRemove() guard passed as prop from parent (centralizes logic)
- Create `src/components/team/AddMemberForm.tsx`:
  - shadcn Command (cmdk v1) inside a Popover for user search/select
  - Role select
  - onMouseDown={(e) => e.preventDefault()} on CommandItem to fix cmdk v1 + Radix Popover focus-loss race
  - Remove controlled value prop from CommandInput (cmdk v1 triggers selectFirstItem on every keystroke with value prop)
  - Calls `POST /api/engagements/:id/team`
- Create `src/components/team/TeamPanel.tsx`: combines TeamMemberTable + AddMemberForm; receives p2Approved prop to control guard logic

### Task 3 — Milestones panel components
- Create `src/hooks/useMilestones.ts`: fetches `GET /api/engagements/:id/milestones`
- Create `src/components/team/MilestoneStatusChip.tsx`: color chip for not_started/on_track/at_risk/complete/overdue
- Create `src/components/team/MilestoneTable.tsx`:
  - 4 milestone rows: planning_approval, evidence_readiness, draft_readiness, final_readiness
  - Date input (DatePicker) per milestone
  - Status chip per milestone (computed)
  - Save button calls `PUT /api/engagements/:id/milestones`
  - try/catch with error toast to prevent blank screen on API failure

### Task 4 — Wire into engagement shell Team tab
