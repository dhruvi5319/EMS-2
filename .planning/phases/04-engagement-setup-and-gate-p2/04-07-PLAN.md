# Plan 04-07: Gate P2 Review UI

**Phase:** 4 — Engagement Setup and Gate P2  
**Status:** not started  
**Features:** F7  

## Goal

Build the Gate P2 Review Panel for QA Reviewers to approve or return the planning baseline, with confirmation dialogs and post-decision state transitions.

## Tasks

### Task 1 — Gate P2 review panel
- Create `src/components/gates/GateP2ReviewPanel.tsx`:
  - Rendered inside `PlanningRecordPanel` when: `isQA && status === 'ready_for_review'`
  - Textarea: rationale (required)
  - "Approve P2" button → calls `POST /api/engagements/:id/gate/p2/approve`
  - "Return for Revision" button → calls `POST /api/engagements/:id/gate/p2/return`
  - Both buttons use AlertDialog with custom async buttons (not AlertDialogAction) to support loading spinner
  - On approve: planning record status updates to approved; GateStatusCard border turns emerald
  - On return: planning record status updates to returned; GateStatusCard border turns amber

### Task 2 — Post-decision navigation and state
- After approve: refresh engagement shell data (gate status cards update)
- After return: planning panel shows returned state; EM can edit and resubmit
- GateStatusCard P2 card reflects new status without page reload

### Task 3 — Playwright E2E tests
- Create `e2e/gate-p2.spec.ts`:
  - Test: QA user sees review panel when planning is ready_for_review
  - Test: non-QA user does not see review panel
  - Test: approve → planning locked, gate card turns green
  - Test: return → planning reverts to draft state, gate card turns amber
  - Test: approve with prerequisites missing → server returns 422, panel shows error

## Acceptance Checks
- [ ] QA sees review panel when planning.status = ready_for_review
- [ ] Non-QA user does not see approve/return buttons
- [ ] Approve requires rationale; updates gate card to approved (emerald)
- [ ] Return requires rationale; updates gate card to returned (amber)
- [ ] Post-approval, planning record shows locked state
- [ ] AlertDialog shows loading spinner during API call
