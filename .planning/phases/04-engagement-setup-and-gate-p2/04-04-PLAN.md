# Plan 04-04: Engagement List + Engagement Shell UI

**Phase:** 4 — Engagement Setup and Gate P2  
**Status:** not started  
**Features:** F4  

## Goal

Build the engagement list page and the tabbed engagement shell page with gate status cards, blocker panel, and metadata editing.

## Tasks

### Task 1 — Engagement list page
- Create `src/hooks/useEngagements.ts`: fetches `GET /api/engagements` with filters
- Create `src/pages/EngagementsPage.tsx`:
  - Filter bar: phase, status, risk_level, owner (selects)
  - Sortable table: job_code, title, phase, owner, risk, gate status summary
  - Row click navigates to `/engagements/:id`
  - CSV export button

### Task 2 — Engagement shell page and gate status cards
- Create `src/components/gates/GateStatusCard.tsx`:
  - Props: gate (A1/P2/P3/P4), status (approved/returned/declined/not_started), approver, date
  - Left border color: emerald (approved), amber (returned), red (declined), slate (not_started)
  - outcome-to-border-color map established as reusable pattern
- Create `src/components/gates/GateStatusCardRow.tsx`: horizontal row of 4 GateStatusCards
- Create `src/components/engagements/BlockerPanel.tsx`: lists open blockers with type and message
- Create `src/pages/EngagementShellPage.tsx`:
  - Tabbed layout (shadcn Tabs): Overview / Team / Planning / Evidence / Findings / Draft / Gate P4 / Audit
  - Overview tab: GateStatusCardRow, BlockerPanel, metadata section (job_code, title, phase, status, risk, owner, portfolio)
  - Edit metadata: inline EditMetadataForm (shadcn Dialog) for EM/AD
  - Other tabs: placeholder until later plans

### Task 3 — Playwright E2E tests
- Create `e2e/engagement-shell.spec.ts`:
  - Test: engagement list loads and filters by phase
  - Test: click row navigates to engagement shell
  - Test: gate status cards show correct border colors
  - Test: edit metadata dialog saves changes

- [ ] Filters work (phase, status, risk, owner)
- [ ] Engagement shell shows 4 gate status cards with correct colors
- [ ] Blocker panel shows open blockers
