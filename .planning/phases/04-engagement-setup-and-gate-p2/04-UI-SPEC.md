## Spacing Scale

Inherited from Phases 1–3 — no changes. Declared here for executor reference:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label padding, badge internal padding |
| sm | 8px | Compact element spacing; field label-to-input gap; char counter gap |
| md | 16px | Default element spacing; card internal padding; form field gap |
| lg | 24px | Section padding; card-to-card gap; planning section separator spacing |
| xl | 32px | Layout gaps; form container vertical padding |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing; top bar height |

Phase 4 exceptions (new additions — no Phase 1/2/3 contradictions):
- Gate status card grid: 4 cards in a horizontal row, `gap-x-16px` (`md`) between cards
- Gate status card min-width: 160px; max-width: 200px; height: auto (content-driven)
- Gate status card left border: 4px solid (color based on gate outcome — see Color section)
- Blocker list item: 16px left padding with 🔴 icon, 8px (`sm`) gap between items
- Planning checklist item: 24px row height, 8px icon-to-text gap
- Milestone table row height: 48px (touch-target compliant, inherits from Phase 3 table rows)
- Independence affirmation switch row: 40px height per team member row (touch target)
- Objective list card: 16px (`md`) internal padding; 8px (`sm`) gap between objective rows
- Revision note banner: 12px vertical padding, 16px horizontal padding (amber banner style)
- Tab bar height: 40px (inherits from Phase 3 `RequestStatusTabs`)
- Touch target minimum: 44×44px on all interactive controls (inherited from Phase 1/2)

## Typography

Inherited from Phases 1–3 — no changes. Declared here for executor reference:

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Form labels, field values, table cells, planning section content, team rows |

| Heading | 20px | 600 (semibold) | 1.2 | Page titles, section headings, engagement shell title, dialog headings |

**Font stack:** `Inter, ui-sans-serif, system-ui, -apple-system, sans-serif`

**Phase 4 typography additions (no contradictions with Phase 1/2/3):**
- Engagement job code (e.g., `ENG-2026-00001`): monospace 14px/600 in the engagement shell header — `ui-monospace, 'Cascadia Code', 'Source Code Pro', monospace` — semibold to anchor the page identity
- Gate card outcome label: 14px/600 — semibold to differentiate from supporting text
- Gate card approver/date: 12px/400 muted — label-role typography
- Milestone status chip: 12px/400 — label role, color-coded background
- P2 checklist item text: 14px/400 — body role; green/red icon precedes text
- Planning section label (e.g., "RISK NOTES"): 12px/400 uppercase tracking-wide — sub-section divider treatment (inherits Phase 3 `FormSectionHeader` pattern)
- Objective text in list: 14px/400, max 2 lines visible (overflow ellipsis, expandable)
- Independence affirmation switch label: 14px/400 — body role
- Locked state revision note textarea: 14px/400 — body role
- Return comment (QA return text): 14px/400 italic in amber-100 background card

### Screen 1: Engagement List Page

**Source:** UX-Mockup-EMS.md — Information Architecture (Engagements nav item)

**Route:** `/engagements`

**Access:** All roles (role-scoped: each user sees only engagements they are assigned to; Admin sees all)

**Layout:**
- Page header: "Engagements" (h1, 20px/600)
- Breadcrumb: "Engagements"
- Top-right: no global CTA (engagements are created only through Gate A1 approval — no manual create button)
- Phase filter tabs below header: All | Planning | Evidence | Draft | Readiness | Closed
- Search input above table: "Search by ID, title, or owner..." (36px height, full-width at mobile)
- Main content: `EngagementTable` with sortable columns

**Engagement table columns:**

| Column | Content | Sortable | Notes |
|--------|---------|----------|-------|
| Job Code | `ENG-YYYY-NNNNN` (monospace 14px/400) | Yes | Click → Engagement Shell |
| Title | Truncated at 60 chars (14px/400) | Yes | Full title in tooltip |
| Phase | `PhaseBadge` (color-coded) | Yes | Planning/Evidence/Draft/Readiness/Closed |
| Owner | Owner display name (14px/400) | Yes | — |
| Risk | `RiskBadge` (Low/Medium/High) | Yes | Inherits risk color from Phase 3 |
| Next Milestone | Label + target date + `MilestoneStatusChip` | Yes | "P3 — Mar 15 ⚠ At Risk" |

| Blocked | `BlockedBadge` ⚠ | No | Shown if open blockers > 0 |

**Default sort:** `updated_at` descending (most recent activity first)

**Table loading state:** 5 `SkeletonRow` components (48px height each)

**Table row click:** Navigates to `/engagements/{id}` (Engagement Shell)

**Table empty state (no engagements):**
- Icon: `Briefcase` (lucide-react, 32px, muted)
- Heading: "No engagements yet." (20px/600)
- Body: "Engagements are created when a request is approved at Gate A1." (14px/400 muted)
- CTA: none (engagements are auto-created; not user-initiated)

**Table empty state (filter active, no results):**
- Icon: `SearchX` (lucide-react, 32px, muted)
- Heading: "No {phase} engagements." (20px/600)
- Body: "Try a different phase filter." (14px/400 muted)
- CTA: `[Clear filter]` (ghost button)

## Visuals

### Visual Hierarchy

Each major surface has one primary visual anchor — the single element that draws the user's eye first and signals the available action.

- **Primary visual anchor (Engagement Shell):** the 4 gate status cards — their color-coded left borders instantly signal engagement health. The `BlockerPanel` below them draws attention via red indicators.
- **Primary visual anchor (Team Tab):** the `[+ Add Member]` accent-fill button and the milestone `[Save Milestones]` primary button at the bottom.
- **Primary visual anchor (Planning Record form):** the P2 readiness checklist at top — users see immediately what is blocking submission. The `[Submit for P2 Review →]` sticky footer CTA is the terminal action.
- **Primary visual anchor (Gate P2 Review):** the P2 completeness checklist (system-computed pass/fail) at top — QA knows instantly whether approval is possible. The `[✓ Approve P2]` accent button anchors the bottom decision panel.

All other interactive elements on these surfaces use secondary (outline), ghost, or muted styles so the primary anchor reads as the clear call-to-action.
