# Plan 04-02: Team + Milestones Backend API

**Phase:** 4 — Engagement Setup and Gate P2  
**Status:** not started  
**Features:** F5  

## Goal

Implement team assignment CRUD and milestone upsert APIs with computed milestone status and P2 prerequisite checking.

## Tasks

### Task 1 — Team assignments API
- Create `src/modules/team/team.repository.ts`, `team.service.ts`, `team.controller.ts`:
  - `GET /api/engagements/:id/team` — list active team assignments with user display_name
  - `POST /api/engagements/:id/team` — add member (EM, AD); validates UNIQUE engagement+user+role
  - `DELETE /api/engagements/:id/team/:assignmentId` — remove member; sets removed_at (EM, AD)
  - Guard: cannot remove last EM from engagement; cannot remove QA if P2 is approved
  - Writes AuditEvent on add/remove

### Task 2 — Milestones API
- Create `src/modules/team/milestones.service.ts` (co-located with team):
  - `GET /api/engagements/:id/milestones` — returns milestone dates + computed status for each
  - `PUT /api/engagements/:id/milestones` — upsert all milestone dates (EM, AD)
  - Milestone status computed on read from target_date vs today (not stored in DB):
    - null target → not_started
    - target > today+7 → on_track
    - target within 7 days → at_risk
    - target passed + gate not approved → overdue
    - gate approved → complete
  - Writes AuditEvent on milestone update

### Task 3 — P2 prerequisites check
- Create `src/modules/gates/p2.prerequisites.ts`:
  - `checkP2Prerequisites(engagementId)` returns `{ met: boolean, missing: string[] }`
  - Checks all 8 prerequisites: objectives≥1, owner set, EM on team, QA on team, milestone≥1 date set, risk_notes non-empty, data_reliability_notes non-empty, independence_status set
- Expose as `GET /api/engagements/:id/gate/p2/prerequisites`
