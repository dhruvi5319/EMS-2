# Plan 04-06: Planning Record Form UI

**Phase:** 4 — Engagement Setup and Gate P2  
**Status:** not started  
**Features:** F6  

## Goal

Build the Planning Record form with objectives accordion, independence affirmation radio, P2 readiness checklist, and locked-state banner for post-approval edits.

## Tasks

### Task 1 — shadcn components for planning
- Manually create `src/components/ui/accordion.tsx` from new-york template
- Manually create `src/components/ui/radio-group.tsx` from new-york template
- Manually create `src/components/ui/textarea.tsx` from new-york template

### Task 2 — Objectives list
- Create `src/components/planning/ObjectiveList.tsx`:
  - Accordion of objectives (shadcn Accordion)
  - Each item: objective_text, information_need (editable inline)
  - "Add Objective" button at bottom → inline form
  - Delete button (soft delete, blocked if evidence linked)

### Task 3 — Planning record form
- Create `src/components/planning/PlanningRecordPanel.tsx`:
  - 4 states: draft / ready_for_review / approved / returned
  - Textarea fields: design_approach, schedule_notes, risk_notes, data_reliability_notes
  - IndependenceStatusRadio component (RadioGroup: affirmed/pending/exception_noted)
  - "Save Draft" button → `PATCH /api/engagements/:id/planning`
  - "Submit for Review" button → `POST /api/engagements/:id/planning/submit`
  - PlanningLockedBanner: shown when status=approved; "Edit (revision required)" button opens revision_note dialog
  - P2ReadinessChecklist (Task 4)
  - isQA prop declared for Plan 04-07 forward compatibility

### Task 4 — P2 readiness checklist
- Create `src/components/planning/P2ReadinessChecklist.tsx`:
  - Fetches `GET /api/engagements/:id/gate/p2/prerequisites`
  - Shows all 8 checks with pass/fail icons
  - refreshTrigger numeric prop to force re-fetch after form saves

### Task 5 — Playwright E2E tests
- Create `e2e/planning.spec.ts`:
  - Test: add objective → appears in accordion
  - Test: fill fields and submit for review → status changes
  - Test: P2 checklist shows correct pass/fail states
