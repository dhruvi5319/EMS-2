# Plan 01-02: Database Migrations

**Phase:** 1 — Foundation  
**Status:** not started  
**Features:** F1  

## Goal

Create all 20 database tables via Knex migrations, matching the exact DDL from TechArch-EMS.md Section 3.2. Includes all indexes and constraints. Database must be fully migrated before any other plan can run.

## Tasks

### Task 1 — Migration 001: Auth tables
Create `src/db/migrations/001_auth_tables.ts`:
- `users` (id, email, display_name, password_hash, is_active, created_at, updated_at)
- `user_roles` (id, user_id FK, role CHECK, assigned_at, assigned_by FK, UNIQUE user_id+role)
- `sessions` (id, user_id FK, token_hash UNIQUE, created_at, expires_at, revoked_at)
- `login_attempts` (id, email, succeeded, ip_address, attempted_at)

### Task 2 — Migration 002: Core domain tables
Create `src/db/migrations/002_core_domain.ts`:
- `requests` (all fields + status CHECK + indexes on status, created_by)
- `engagements` (all fields + phase/status/risk_level CHECKs + indexes on owner, phase, status)
- `team_assignments` (all fields + UNIQUE engagement_id+user_id+role)
- `milestones` (all fields + UNIQUE engagement_id)
- `planning_records` (all fields + independence_status/status CHECKs + UNIQUE engagement_id)
- `objectives` (all fields + status CHECK + indexes on planning_record_id, engagement_id + deleted_at)
- `planning_revisions` (all fields)
- `evidence_items` (all fields + evidence_type/sensitivity CHECKs + indexes + deleted_at)
- `evidence_files` (all fields + deleted_at)
- `objective_evidence_links` (all fields + UNIQUE objective_id+evidence_item_id)
- `findings` (all fields + status CHECK + deleted_at)
- `finding_evidence_links` (all fields + UNIQUE finding_id+evidence_item_id)
- `draft_products` (all fields + status CHECK + UNIQUE engagement_id)
- `draft_statements` (all fields + reference_status CHECK + indexes on draft_product_id, ref_status)
- `statement_evidence_links` (all fields + UNIQUE statement_id+evidence_item_id)
- `draft_comments` (all fields — append-only)
- `gate_decisions` (all fields + gate_type/status CHECKs + indexes on engagement_id, gate_type)
- `audit_events` (all fields + indexes on engagement_id, actor_id, action, occurred_at DESC)

## Acceptance Checks
- [ ] `knex migrate:latest` runs without errors
- [ ] All 20 tables exist with correct columns, types, indexes, and constraints
- [ ] Migration is idempotent (running twice does not error)
