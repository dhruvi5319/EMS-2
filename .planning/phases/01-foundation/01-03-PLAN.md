# Plan 01-03: Auth API + RBAC Middleware + Seed

**Phase:** 1 — Foundation  
**Status:** not started  
**Features:** F0 (partial — auth/session/user infrastructure)  

## Goal

Implement login/logout/me endpoints with session-cookie hybrid auth, account lockout, requireRole RBAC middleware, and an admin seed user. The system must be able to authenticate and authorize before any other feature is built.

## Tasks

### Task 1 — Auth service and routes
- Create `src/modules/auth/auth.repository.ts`: getUserByEmail, createSession, getSessionByHash, revokeSession, recordLoginAttempt, countRecentFailures
- Create `src/modules/auth/auth.service.ts`:
  - `login(email, password)` → validates credentials, checks lockout (5 failed in 15 min), creates session, returns JWT token
  - `logout(tokenHash)` → revokes session
  - `me(tokenHash)` → returns current user with roles
- Create `src/modules/auth/auth.controller.ts`:
  - `POST /api/auth/login` → 200 with token + user, or 401/423 (locked)
  - `POST /api/auth/logout` → 204
  - `GET /api/auth/me` → 200 with user or 401
- Mount auth router in `src/routes/index.ts`

### Task 2 — RBAC middleware
- Create `src/middleware/auth.ts`: extract Bearer token from header, validate session hash in DB, attach `req.user` (id, email, display_name, roles)
- Create `src/middleware/rbac.ts`: `requireRole(...roles)` factory — returns 401 if unauthenticated, 403 if role not in user.roles
- Apply `requireAuth` middleware globally to all `/api/*` routes except `/api/health` and `/api/auth/*`

### Task 3 — Seed script
- Create `src/db/seeds/001_admin.ts`:
  - Creates admin user (email: admin@ems.local, password: Admin1234!, roles: [AD, AL, EM, AN, QA, IR, PC, RO])
  - Creates test users for each role (al@ems.local, em@ems.local, an@ems.local, qa@ems.local, ir@ems.local, pc@ems.local, ro@ems.local — password: Test1234!)
  - Uses bcryptjs cost 12
  - Idempotent (uses INSERT ... ON CONFLICT DO NOTHING)

## Acceptance Checks
- [ ] `POST /api/auth/login` with valid credentials returns token and user object
- [ ] `POST /api/auth/login` with invalid credentials 5 times locks the account (returns 423)
- [ ] `GET /api/auth/me` with valid token returns current user with roles
- [ ] `GET /api/auth/me` without token returns 401
- [ ] Any protected route without token returns 401
- [ ] Any protected route with insufficient role returns 403
- [ ] Seed users exist and can log in
