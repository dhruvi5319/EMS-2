# Plan 01-04: Login UI + App Shell

**Phase:** 1 — Foundation  
**Status:** not started  
**Features:** F0 (partial — login UI and authenticated shell)  

## Goal

Build the React login page, AuthContext for session persistence, and the AppShell layout (sidebar + topbar) that all future pages will live inside. Users should be able to log in and see the authenticated shell.

## Tasks

### Task 1 — Auth context and login page
- Create `src/auth/AuthContext.tsx`:
  - Stores current user (id, email, display_name, roles) and token in memory
  - On mount, calls `GET /api/auth/me` to restore session (eliminates re-login on page reload)
  - Provides `login(email, password)` and `logout()` methods
- Create `src/auth/ProtectedRoute.tsx`: redirects unauthenticated users to `/login`
- Create `src/pages/LoginPage.tsx`:
  - Form: email, password, submit button
  - Shows error on invalid credentials or account lockout
  - On success, navigates to `/dashboard`
  - Plain Tailwind CSS (no shadcn at this stage)

### Task 2 — App shell layout
- Create `src/layout/AppShell.tsx`:
  - Sidebar with nav links (placeholder — role filtering added in Phase 2)
  - Topbar with app name and logout button
  - Main content area (children / Outlet)
- Create `src/pages/DashboardPage.tsx`: placeholder "Dashboard" page
- Wire routes in `src/App.tsx`:
  - `/login` → LoginPage (public)
  - `/*` → ProtectedRoute → AppShell → page routes

### Task 3 — Playwright E2E smoke test
- Create `e2e/auth.spec.ts`:
  - Test: unauthenticated visit to `/` redirects to `/login`
  - Test: login with admin credentials navigates to dashboard
  - Test: logout returns to login page

## Acceptance Checks
- [ ] Visiting `/` while unauthenticated redirects to `/login`
- [ ] Login with valid credentials shows the AppShell dashboard
- [ ] Login with invalid credentials shows an error message
- [ ] Logout clears session and returns to login page
- [ ] Page reload while authenticated stays on current page (session restored via /api/auth/me)
