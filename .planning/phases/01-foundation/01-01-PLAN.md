# Plan 01-01: Docker + Project Scaffold

**Phase:** 1 — Foundation  
**Status:** not started  
**Features:** F0 (partial), F1  

## Goal

Stand up the complete 4-service docker-compose stack (frontend, backend, database, storage) with a working Node.js/Express/TypeScript backend and a React/Vite/Tailwind frontend. No business logic — only project structure, build tooling, and container wiring.

## Tasks

### Task 1 — Backend scaffold
- Initialize `backend/` as a Node.js TypeScript project
- Install: express, typescript, ts-node, knex, pg, bcryptjs, zod, jsonwebtoken, multer, cors, dotenv
- Install dev: @types/*, jest, supertest, ts-jest
- Configure `tsconfig.json`, `knexfile.ts`, `jest.config.ts`
- Create `src/app.ts` (Express app, CORS, JSON body parser, health route `GET /api/health`)
- Create `src/server.ts` (HTTP listen entry point)
- Create `src/config/env.ts` (requireEnv() fail-fast validation for DATABASE_URL, SESSION_SECRET)
- Create `src/middleware/logger.ts` (request logging)
- Create `src/shared/errors.ts` (AppError class)
- Create `src/db/connection.ts` (Knex PostgreSQL connection pool)
- Create `backend/Dockerfile` (node:20-alpine, build + production stage)

### Task 2 — Frontend scaffold
- Initialize `frontend/` with Vite + React + TypeScript template
- Install: react-router-dom, @tanstack/react-query, axios, tailwindcss, postcss, autoprefixer
- Configure `tailwind.config.ts` with CSS variable token system
- Create `src/main.tsx`, `src/App.tsx` (router root, placeholder routes)
- Create `src/api/client.ts` (axios wrapper, auth token injection, 401 redirect)

### Task 3 — docker-compose
- Create `docker-compose.yml` with 4 services:
  - `db`: postgres:15-alpine, volume for data persistence
  - `storage`: local volume at `./data/files`
  - `app`: backend container (depends on db)
  - `frontend`: node:20-alpine dev server (port 5173)
- Create `.env.example` with all required environment variables
- Verify `docker-compose up` starts all services without errors

## Acceptance Checks
- [ ] `docker-compose up` starts all 4 services without errors
- [ ] `GET /api/health` returns `{ status: "ok" }` with HTTP 200
- [ ] Frontend dev server serves the React app at `http://localhost:5173`
- [ ] Backend connects to PostgreSQL (no connection errors on startup)
