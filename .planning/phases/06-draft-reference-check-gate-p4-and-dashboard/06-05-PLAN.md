8px segmented bar construction:
- Total width: 100%; height: 8px (h-2); rounded-full container; overflow-hidden
- Segments in order: Passed (accent blue-600) → Waived (gray-300) → Failed (red-600) → In Review (yellow-500) → Not Started (gray-100)
- Width of each segment: `${(count / total) * 100}%`; segments rendered as adjacent flex children (no gap)
- If total=0: render single gray-100 bar (empty state)

Progress label (above bar, right-aligned):
- "N statements" (14px/400 left) + `(Z%)` (12px/400 muted right) on same row (flex justify-between)

Count row (below bar):
- "Passed: N · Waived: N · Failed: N · In Review: N · Not Started: N" (12px/400 muted)

P4GateStatusLine (from UI-SPEC):
- Blocked: `<XCircle size={16} className="text-red-700 mr-1" />` + "P4 Status: BLOCKED — N Failed, N In Review" (text-red-700, text-sm)
- Ready: `<CheckCircle size={16} className="text-green-800 mr-1" />` + "P4 Status: READY — All checks complete" (text-green-800, text-sm)
- Ready condition: failed === 0 AND inReview === 0 AND notStarted === 0

Accessibility:
- `role="progressbar"` on segment bar container
- `aria-valuenow={passed + waived}` + `aria-valuemin={0}` + `aria-valuemax={total}` + `aria-label="Reference check progress: {N} of {M} complete"`
- Each segment: `role="img"` + `aria-label="{Status}: {N} statements"`
