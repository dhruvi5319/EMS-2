# Plan 06-05: Statements / Indexing UI (AN/EM View)

**Phase:** 6 — Draft Product, Reference Check, Gate P4, and Dashboard  
**Status:** not started  
**Features:** F12  

## Goal

Build the Statements (Indexing) page for Analysts and Engagement Managers — adding statements, linking evidence, and viewing the 5-segment progress bar.

## Tasks

### Task 1 — Statements hook
- Create `src/hooks/useStatements.ts`: fetches `GET /api/engagements/:id/statements`

### Task 2 — Statements page (AN/EM view)
- Create `src/pages/StatementsPage.tsx` (route: `/engagements/:id/statements`):
  - 5-segment progress bar:
    - Segments: not_started / in_review / passed / failed / waived
    - Colors per segment per UI-SPEC
  - Statement table:
    - Columns: # (sort_order), statement text (truncated), reference_status badge, linked evidence count, IR assignee, actions
    - ReferenceStatusBadge: uses inline className per UI-SPEC color table (not shadcn Badge variants)
    - Row click navigates to statement detail
  - "Add Statement" button → AddStatementForm

### Task 3 — Add statement form
- Create `src/components/statements/AddStatementForm.tsx`:
  - Textarea: statement_text (required)
  - Multi-select evidence items (cmdk Command in Popover)
    - onMouseDown={(e) => e.preventDefault()} on CommandItem: canonical cmdk v1 + Radix Popover focus-loss fix
  - Calls `POST /api/engagements/:id/statements`

### Task 4 — Waive dialog
- Create `src/components/statements/WaiveDialog.tsx`:
  - EM/AD only
  - Textarea: discrepancy_notes (required justification)
  - Calls `PATCH /api/engagements/:id/statements/:id` with reference_status=waived
  - Non-EM/AD sees disabled waive button with tooltip

- [ ] Add statement with evidence links works
