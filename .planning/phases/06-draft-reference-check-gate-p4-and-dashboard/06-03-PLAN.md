# Plan 06-03: Gate P4 Backend API + Portfolio List/Export API

**Phase:** 6 — Draft Product, Reference Check, Gate P4, and Dashboard  
**Status:** not started  
**Features:** F13, F14  

## Goal

Implement Gate P4 prerequisites check and approval, plus the portfolio engagement list and CSV export APIs for the dashboard.

## Tasks

### Task 1 — Gate P4 prerequisites and decision
- Create `src/modules/gates/p4.prerequisites.ts`:
  - `checkP4Prerequisites(engagementId)`:
    1. P3 gate has status=approved
    2. No draft_statement has reference_status = in_review or failed (all must be passed or waived)
    3. No open blockers
  - Note: statements queried via draft_products FK join (no direct engagement_id on statements)
- Add to gates routes (PC, AD):
  - `GET /api/engagements/:id/gate/p4/prerequisites`
  - `POST /api/engagements/:id/gate/p4/approve`:
    - Server-side re-validates P4 prerequisites
    - `outcome` field: 'ready_for_issuance' or 'closed'
    - 'ready_for_issuance' → engagement.phase='readiness', engagement.status='ready_for_issuance'
    - 'closed' → engagement.phase='closed', engagement.status='closed'
    - Creates GateDecision (gate_type=P4, status=approved)
    - Creates AuditEvent (GATE_P4_APPROVED)

### Task 2 — Portfolio list and export API
- Add to engagements service/routes:
  - `GET /api/engagements` (enhanced) — supports filters: phase, status, owner_id, risk_level, due_date range
  - `GET /api/engagements/export?format=csv` — CSV export:
    - Registered BEFORE /:id route to avoid parameterized route capture
    - Columns: job_code, title, phase, status, risk_level, owner_name, next_milestone, gate_status_summary
    - Access: canExport allowlist [AD, EM, AN, QA, AL, PC, RO] — excludes IR
    - Uses 4 parallel phase-filtered queries for accurate phase counts (dashboard summary)
  - `GET /api/engagements/summary` — phase/status counts for stat cards
