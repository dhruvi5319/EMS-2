**checkP4Prerequisites:**
```typescript
export async function checkP4Prerequisites(engagementId: string): Promise<{
  met: boolean;
  blockers: Array<{ type: string; message: string; statement_ids?: string[] }>;
}>
```
Four checks in order:
1. **p3_not_approved**: `SELECT * FROM gate_decisions WHERE engagement_id = :id AND gate_name = 'P3' AND decision = 'approved'` — if none: blocker "Gate P3 has not been approved."
2. **has_failed_checks**: `SELECT id FROM draft_statements WHERE engagement_id = :id AND ref_status = 'failed'` — if any: blocker "N failed reference check(s)." with statement_ids[]
3. **has_in_review_checks**: `SELECT id FROM draft_statements WHERE engagement_id = :id AND ref_status = 'in_review'` — if any: blocker "N check(s) still In Review." with statement_ids[]
4. **has_not_started_checks**: `SELECT id FROM draft_statements WHERE engagement_id = :id AND ref_status = 'not_started'` — if any: blocker "N check(s) not yet started." with statement_ids[]

`met` is `true` only when all four checks pass (blockers array is empty).

**recordP4Decision:**
```typescript
export async function recordP4Decision(
  engagementId: string,
  data: { decision: 'approved', outcome: 'ready_for_issuance' | 'closed', comment?: string },
  userId: string
): Promise<GateDecision>
```
- Re-run `checkP4Prerequisites` inside the function; throw 422 if not met (server-side re-validation per security pattern used in P2/P3)
- In a single Knex transaction:
  1. `INSERT INTO gate_decisions (id, engagement_id, gate_name, decision, decided_by, comment, decided_at) VALUES (uuid, :engId, 'P4', 'approved', :userId, :comment, NOW())`
  2. `INSERT INTO audit_events (id, engagement_id, actor_id, action_code, target_type, target_id, metadata, created_at) VALUES (uuid, :engId, :userId, 'GATE_P4_APPROVED', 'engagement', :engId, JSON.stringify({ outcome }), NOW())`
  3. `UPDATE engagements SET status = :outcome, phase = (:outcome === 'ready_for_issuance' ? 'readiness' : 'closed'), updated_at = NOW() WHERE id = :engId`
- Return the created GateDecision record
- GateDecision is immutable — no update/delete after creation

**listEngagements (for portfolio dashboard F14):**
```typescript
export async function listEngagements(filters: {
  phase?: string;
  status?: string;
  owner_id?: string;
  risk_level?: string;
  due_date_before?: string;
  sort_by?: string;
  sort_dir?: 'asc' | 'desc';
  limit?: number;
  offset?: number;
}, viewerRoles?: string[]): Promise<{ engagements: EngagementListItem[], total: number }>
```
- Base query: `SELECT e.*, u.display_name as owner_name FROM engagements e LEFT JOIN users u ON e.owner_id = u.id`
- Apply filters: WHERE conditions for phase, status, owner_id, risk_level; `due_date <= due_date_before` if provided
- Sort: map sort_by values to column names; default `updated_at DESC`
- Pagination: LIMIT/OFFSET; separate COUNT(*) for total
- Join gate_decisions for each gate (A1/P2/P3/P4) to include gate_a1_status, gate_p2_status, gate_p3_status, gate_p4_status:
  ```sql
  LEFT JOIN gate_decisions gd_a1 ON e.id = gd_a1.engagement_id AND gd_a1.gate_name = 'A1' AND gd_a1.decision = 'approved'
  LEFT JOIN gate_decisions gd_p2 ON e.id = gd_p2.engagement_id AND gd_p2.gate_name = 'P2' AND gd_p2.decision = 'approved'
  LEFT JOIN gate_decisions gd_p3 ON e.id = gd_p3.engagement_id AND gd_p3.gate_name = 'P3' AND gd_p3.decision = 'approved'
  LEFT JOIN gate_decisions gd_p4 ON e.id = gd_p4.engagement_id AND gd_p4.gate_name = 'P4' AND gd_p4.decision = 'approved'
  ```
  Gate status: 'approved' if join matches, else compute: any pending gate_decision → 'pending', else 'not_started'
- Join milestones to find next_milestone (earliest incomplete milestone by target_date)
- has_blockers: subquery checking if any open blockers exist for engagement
