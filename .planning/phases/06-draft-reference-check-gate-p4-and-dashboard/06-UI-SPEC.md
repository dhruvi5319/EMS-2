# Phase 6 — UI Design Contract

> Visual and interaction contract for Phase 6: Draft Product, Reference Check, Gate P4, and Dashboard.
> Generated by pivota_spec-ui-researcher, verified by pivota_spec-ui-checker.
>
> **Phase 6 UI Scope:** Draft Product page (F11), Draft Statements / Indexing panel (F12), Reference
> Check Queue for IR (F12), Reference Check progress indicator (F12), Gate P4 Final Readiness page
> (F13), Portfolio Dashboard (F14), and all associated empty states, confirmation dialogs, and role-
> gated views. This is the FINAL phase — it completes the full engagement lifecycle from intake to
> final readiness.
>
> **Critical constraint:** This spec EXTENDS Phases 1–5 — all spacing, typography, color, layout
> constants, and design system tokens are LOCKED and must not be re-declared with different values.
> Phase 6 introduces new shadcn components, new screen layouts, new interaction patterns, and new
> semantic colors for reference check status and engagement terminal states without contradicting
> any prior phase.

## Spacing Scale

Inherited from Phases 1–5 — no changes. Declared here for executor reference:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label padding, badge internal padding |
| sm | 8px | Compact element spacing; field label-to-input gap; char counter gap |
| md | 16px | Default element spacing; card internal padding; form field gap |
| lg | 24px | Section padding; card-to-card gap; draft form section separator spacing |
| xl | 32px | Layout gaps; form container vertical padding |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing; top bar height |

Phase 6 exceptions (new additions — no Phase 1–5 contradictions):
- Draft status stepper width: full-width minus card padding (`md` each side); step nodes: 24px diameter circles
- Draft status stepper connector line height: 2px; gap between step node and label: 4px (`xs`)
- Review comment thread item: min-height 72px (name + date line + comment text); padding: 16px (`md`)
- Comment input textarea min-height: 80px (≈3 rows at 14px line-height 1.5)
- Statement list row height: 48px (touch-target compliant)
- Reference check progress bar height: 8px (matches Phase 5 coverage bar pattern)
- Reference check progress bar status segments: 5 colored segments (Passed/Waived/Failed/In Review/Not Started)
- Statement detail discrepancy panel: left border 4px solid red-600 (signals failed state — distinct from accent panel pattern)
- Waiver modal max-width: 520px
- Statement edit view discrepancy card: padding 16px (`md`), left border 4px solid amber-500
- P4 prerequisites checklist item row: 40px min-height (touch-target) with 8px icon-to-text gap
- Gate P4 approval panel: same layout as P2/P3 panels — left border 4px accent, padding 24px
- Portfolio dashboard stat cards: 4 cards row (Planning/Evidence/Draft/Readiness), height 80px, gap 16px (`md`)
- Portfolio dashboard table row height: 48px (touch-target compliant)
- Gate mini-status chips in dashboard: 20px diameter circle per gate (A1/P2/P3/P4); 4px gap between gates
- CSV export button height: 36px (secondary action, matches Phase 5 Export CSV pattern)
- Draft create form max-width: 480px (modal dialog)
- Touch target minimum: 44×44px on all interactive controls (inherited)

**Phase 6 additional semantic colors — Reference Check Status:**

These are DISTINCT from Phase 5 P3 sufficiency chips and Phase 4 milestone chips. Used exclusively
for draft statement reference check status throughout Phase 6.

| Status | Background | Text | Border | Usage |
|--------|-----------|------|--------|-------|
| Not Started | `hsl(0 0% 93%)` — gray-100 | `hsl(0 0% 35%)` — gray-600 | none | Statement reference status badge |
| In Review | `hsl(45 96% 88%)` — yellow-100 | `hsl(45 90% 28%)` — yellow-800 | none | Statement reference status badge |
| Passed | `hsl(142 71% 88%)` — green-100 | `hsl(142 70% 28%)` — green-800 | `hsl(142 71% 45%)` — emerald-600 1px | Statement reference status badge |
| Failed | `hsl(0 72% 93%)` — red-100 | `hsl(0 72% 38%)` — red-700 | `hsl(0 72% 51%)` — red-600 1px | Statement reference status badge |
| Waived | `hsl(0 0% 93%)` — gray-100 | `hsl(0 0% 35%)` — gray-600 | `hsl(0 0% 70%)` — gray-300 1px dashed | Statement reference status badge |

> Note: Passed and Failed carry a 1px colored border (consistent with Phase 5 P3 sufficiency chip
> border treatment). Not Started and In Review match Phase 4 milestone chip style (no border).
> Waived uses a dashed gray border — distinct from all other status types — visually signaling
> intentional skip rather than outcome.
