# Plan 06-01: Draft Product Backend API

**Phase:** 6 — Draft Product, Reference Check, Gate P4, and Dashboard  
**Status:** not started  
**Features:** F11  

## Goal

Implement the draft product CRUD backend with file upload/replace, status transitions, and append-only review comments.

## Tasks

### Task 1 — Draft product repository and service
- Create `src/modules/draft/draft.repository.ts`:
  - getDraft(engagementId) — UNIQUE per engagement
  - createDraft(data)
  - updateDraft(engagementId, data)
  - addComment(draftProductId, authorId, authorName, commentText)
  - getComments(draftProductId)
- Create `src/modules/draft/draft.service.ts`:
  - `getDraft(engagementId)` — creates if not exists; returns with comments
  - `updateDraft(engagementId, data, actorId)` — validates allowed status transitions:
    - drafting → under_review (EM, AN, AD)
    - under_review → drafting (QA can return to drafting; enforced at route layer)
    - under_review → ready_for_reference_check (QA, EM, AD)
    - ready_for_reference_check → ready_for_final_review (EM, PC, AD)
  - `uploadFile(engagementId, buffer, mimeType, filename, actorId)` — stores via StorageProvider; updates draft_file_ref/draft_filename; calls getDraft() after upload to return consistent shape
  - `addComment(engagementId, actorId, authorName, commentText)` — append-only
  - Writes AuditEvent on status changes

### Task 2 — Draft routes
- Create `src/modules/draft/draft.controller.ts` mounted at `/api/engagements/:id/draft`:
  - `GET /api/engagements/:id/draft` — get or create
  - `PATCH /api/engagements/:id/draft` — update fields (title, version, owner_id, status, review_comments)
  - `POST /api/engagements/:id/draft/file` — upload file (multer memory; ≤25 MB); returns `{ draft: DraftProduct }` shape
  - `GET /api/engagements/:id/draft/file` — stream file download
  - `POST /api/engagements/:id/draft/comments` — add comment (append-only)

## Acceptance Checks
- [ ] Draft product created on first GET; returns consistent shape including comments
- [ ] Status transitions enforced (invalid transitions return 422)
- [ ] File upload updates draft_file_ref and draft_filename
- [ ] `POST /draft/file` returns `{ draft: DraftProduct }` shape (not just success)
- [ ] Comments are append-only; GET returns full comment history
- [ ] AuditEvent written on status changes
