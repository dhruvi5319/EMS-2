# Plan 06-02: Statements + Reference Check Backend API

**Phase:** 6 — Draft Product, Reference Check, Gate P4, and Dashboard  
**Status:** not started  
**Features:** F12  

## Goal

Implement draft statement CRUD with evidence multi-linking, reference status management, waiver support, and discrepancy assignment.

## Tasks

### Task 1 — Statements repository and service
- Create `src/modules/statements/statements.repository.ts`:
  - listStatements(draftProductId) — via draft_products FK bridge (no direct engagement_id on statements)
  - getStatement(statementId)
  - createStatement(draftProductId, data)
  - updateStatement(statementId, data)
  - linkEvidence(statementId, evidenceItemIds, actorId)
  - unlinkEvidence(statementId, evidenceItemId)
- Create `src/modules/statements/statements.service.ts`:
  - `listStatements(engagementId)` — resolves draft_product_id from engagement, then queries statements
  - `createStatement(engagementId, data, actorId)` — creates statement; optionally links evidence
  - `updateStatement(statementId, data, actorId, userRoles)`:
    - reference_status updates: IR can set in_review/passed/failed; requires discrepancy_notes when failed
    - Waiver: EM or AD only; requires discrepancy_notes as justification (throws 403 for AN/IR, 422 if notes empty)
    - Discrepancy assignment: sets assigned_to_analyst
    - Writes AuditEvent: STATEMENT_DISCREPANCY_ASSIGNED when assigned_to_analyst set
    - Uses action/object_type/after_state pattern for audit events

### Task 2 — Statements routes
- Create `src/modules/statements/statements.controller.ts`:
  - `GET /api/engagements/:id/statements` — list all statements with linked evidence
  - `POST /api/engagements/:id/statements` — create (AN, EM, AD)
  - `PATCH /api/engagements/:id/statements/:statementId` — update (AN, EM, IR, AD)
  - `POST /api/engagements/:id/statements/:statementId/evidence` — link evidence (AN, EM, AD)
  - `DELETE /api/engagements/:id/statements/:statementId/evidence/:evidenceId` — unlink

## Acceptance Checks
- [ ] Create statement with evidence links works
- [ ] IR can update reference_status to in_review/passed/failed
- [ ] Failed status requires discrepancy_notes (422 if missing)
- [ ] Waiver restricted to EM/AD; requires discrepancy_notes (403 for others; 422 if notes empty)
- [ ] Discrepancy assignment writes AuditEvent
- [ ] Statement list resolves correctly via draft_products FK bridge
