# Plan 06-07: Gate P4 UI + Portfolio Dashboard UI

**Phase:** 6 — Draft Product, Reference Check, Gate P4, and Dashboard  
**Status:** not started  
**Features:** F13, F14  

## Goal

Build the Gate P4 review tab embedded in the Engagement Shell and the Portfolio Dashboard page with stat cards, sortable engagement table, and CSV export.

## Tasks

### Task 1 — Gate P4 review UI (embedded in EngagementShellPage)
- Create `src/components/gates/GateP4ReviewPage.tsx` (rendered as TabsContent in EngagementShellPage, NOT a navigate-away page; useParams() works at /engagements/:id route context):
  - P4 prerequisites checklist (reuses pattern from P2/P3)
  - Outcome select: ready_for_issuance / closed (PC, AD only)
  - Rationale textarea (required)
  - "Approve P4" button → AlertDialog confirmation → `POST /api/engagements/:id/gate/p4/approve`
  - P4DecisionPanel follows identical 4px accent left border pattern as P2/P3 panels
  - On approval: engagement status and phase update; GateStatusCard P4 turns emerald

### Task 2 — Add Gate P4 tab to EngagementShellPage
- Add "Gate P4" TabsTrigger and TabsContent to `EngagementShellPage.tsx`
- Render `<GateP4ReviewPage>` inside TabsContent
- Wire P4 gate status from engagement detail response to the P4 GateStatusCard

### Task 3 — Portfolio Dashboard page (replaces DashboardPage stub)
- Create `src/hooks/usePortfolio.ts`:
  - fetchPhaseCounts: uses 4 parallel phase-filtered queries (limit=1) for accurate counts
  - exportCSV: uses fetch blob → createObjectURL → anchor.click() pattern
- Create `src/pages/PortfolioDashboardPage.tsx` (replaces DashboardPage at `/dashboard`):
  - Stat cards: count by phase (planning / evidence / draft / readiness / closed)
  - Filter bar: owner, risk_level, due_date range, phase, status
  - Sortable engagement register table: job_code, title, phase, owner, risk, next_milestone, gate_status
  - CSV export button (allowlist: [AD, EM, AN, QA, AL, PC, RO]; IR excluded)
