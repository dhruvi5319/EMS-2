# Plan 06-04: Draft Product UI

**Phase:** 6 — Draft Product, Reference Check, Gate P4, and Dashboard  
**Status:** not started  
**Features:** F11  

## Goal

Build the Draft Product page with status stepper, file upload section, review comments thread, and status transition controls.

## Tasks

### Task 1 — Draft product hook
- Create `src/hooks/useDraftProduct.ts`:
  - Fetches `GET /api/engagements/:id/draft`
  - `updateDraft(data)` — calls PATCH
  - `uploadFile(file)` — uses raw fetch() with FormData (NOT api.ts wrapper — JSON Content-Type breaks multipart)
    - After upload: resets uploadFile state to avoid stale file reference
  - `addComment(text)` — calls POST /comments

### Task 2 — Draft product page
- Create `src/pages/DraftProductPage.tsx` (route: `/engagements/:id/draft`):
  - DraftStatusStepper: pure Tailwind (no shadcn stepper); 4-step nodes with CheckCircle for completed, number for active/upcoming; steps: Drafting → Under Review → Ready for Reference Check → Ready for Final Review
  - Metadata section: title, version, owner (editable by EM/AN/AD)
  - Status transition buttons (based on current status and user role):
    - EM/AN/AD: Drafting → Under Review
    - QA: Under Review → Drafting (return to drafting)
    - QA/EM/AD: Under Review → Ready for Reference Check
    - EM/PC/AD: Ready for Reference Check → Ready for Final Review
  - File section: drag-and-drop upload + current file download link
  - Review Comments: append-only thread (AddCommentForm + CommentList)

## Acceptance Checks
- [ ] DraftStatusStepper shows correct active step with CheckCircle for completed steps
- [ ] Status transitions only available to authorized roles
- [ ] File upload works; state resets after upload (no stale file reference)
- [ ] Review comments appear in chronological order; new comments append
- [ ] Draft created on first visit if it doesn't exist
