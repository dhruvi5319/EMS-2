# Plan 06-06: Statement Detail UI (IR + AN Views)

**Phase:** 6 — Draft Product, Reference Check, Gate P4, and Dashboard  
**Status:** not started  
**Features:** F12  

## Goal

Build the Statement Detail page with IR reference check decision panel, discrepancy fields, and Analyst correction view.

## Tasks

### Task 1 — Statement detail page
- Create `src/pages/StatementDetailPage.tsx` (route: `/engagements/:id/statements/:statementId`):
  - Uses GET list endpoint + client-side filter (no single-statement GET in backend; avoids extra API)
  - Header: statement text, sort order, reference_status badge
  - Linked evidence: list of linked evidence items with links
  - IR Decision Panel (Task 2) — shown to IR users
  - Analyst Correction Notice (Task 3) — shown to AN users when discrepancy assigned
  - Team info: note that team API shape is `{assignments}` grouped by user_id for multi-role users

### Task 2 — IR decision panel
- Create `src/components/statements/IRDecisionPanel.tsx` (IR users only):
  - Reference status select: not_started / in_review / passed / failed
  - Discrepancy notes textarea (required when status = failed)
  - Assign to Analyst select (when status = failed)
  - "Save Decision" button → `PATCH /api/engagements/:id/statements/:id`

### Task 3 — Analyst correction notice
- Create `src/components/statements/AnalystCorrectionNotice.tsx` (AN users when assigned_to_analyst = current user):
  - Shows discrepancy_notes from IR
  - "Mark Revision Ready" button → sets revision_ready=true via PATCH

### Task 4 — Discrepancy panel
- Create `src/components/statements/DiscrepancyPanel.tsx`:
  - Shows current discrepancy state (notes, assigned analyst, revision_ready flag)
  - Visible to EM/QA/IR/AD

## Acceptance Checks
- [ ] Statement detail shows correct statement text and linked evidence
- [ ] IR user sees decision panel with status select and discrepancy notes
- [ ] Setting status=failed requires discrepancy_notes
- [ ] AN user sees correction notice when discrepancy assigned to them
- [ ] Mark revision ready updates revision_ready flag
- [ ] Non-IR/AN users see DiscrepancyPanel in read-only mode
