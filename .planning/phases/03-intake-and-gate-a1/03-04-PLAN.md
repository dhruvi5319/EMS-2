# Plan 03-04: Request Detail Page + Intake File Upload UI

**Phase:** 3 — Intake and Gate A1  
**Status:** not started  
**Features:** F2  

## Goal

Build the request detail page with all metadata, intake document section with drag-and-drop upload, and the Gate A1 decision panel slot.

## Tasks

### Task 1 — Request detail page
- Create `src/hooks/useRequest.ts`: fetches `GET /api/requests/:id`
- Create `src/pages/RequestDetailPage.tsx`:
  - Header: request type badge, topic, status badge, created date
  - Metadata section: requester, agency/program, due date, notes
  - Intake document section (see Task 2)
  - Gate A1 decision slot: `data-section="gate-a1"` placeholder (populated by Plan 03-05)
  - Edit button (navigates to request form if status = draft)

### Task 2 — IntakeFileUpload component
- Create `src/components/requests/IntakeFileUpload.tsx`:
  - Drag-and-drop zone + file picker button
  - Accepts: PDF, DOCX, DOC, XLSX, XLS, CSV, TXT, PNG, JPG (≤25 MB)
  - Shows upload progress (fetch with ReadableStream or XMLHttpRequest progress events)
  - On re-upload, warns user that previous file will be replaced
  - Uses raw `fetch()` with FormData (NOT the api.ts wrapper — which forces JSON Content-Type)
  - Calls `POST /api/requests/:id/upload`
  - On success: shows filename, file size, download link
  - Error states: file too large, invalid type, network error

### Task 3 — Intake document download
- Wire `GET /api/requests/:id/file` to a download link in the IntakeFileUpload component

## Acceptance Checks
- [ ] Request detail page shows all metadata fields
- [ ] Drag-and-drop and click-to-upload both work
- [ ] Progress indicator shows during upload
- [ ] Re-upload replaces previous file
- [ ] File too large shows error (>25 MB)
- [ ] Download link retrieves the uploaded file
- [ ] Status badge reflects current request status
