# Plan 03-03: Request List + New/Edit Request Form UI

**Phase:** 3 — Intake and Gate A1  
**Status:** not started  
**Features:** F2  

## Goal

Build the request list page with status tabs and sortable table, plus the new/edit request form with validation and draft/submit flow.

## Tasks

### Task 1 — shadcn components for requests
- Manually create `src/components/ui/tabs.tsx` from new-york template
- Manually create `src/components/ui/form.tsx` (react-hook-form integration) from new-york template
- Manually create `src/components/ui/popover.tsx` from new-york template
- Manually create `src/components/ui/calendar.tsx` (react-day-picker v10) from new-york template

### Task 2 — Request list page
- Create `src/hooks/useRequests.ts`: fetches `GET /api/requests`
- Create `src/pages/RequestsPage.tsx`:
  - Status tabs: All / Draft / Submitted / Accepted / Declined
  - Sortable table: request type, requester, topic, due date, status, created date
  - "New Request" button (navigates to form)
  - Each row links to request detail

### Task 3 — New/Edit request form
- Create `src/pages/RequestFormPage.tsx` (used for both create and edit):
  - react-hook-form with Zod validation schema
  - Fields: request_type (Select), requester, topic, agency_program, due_date (DatePicker), notes
  - "Save as Draft" button → creates/updates without validation
  - "Submit" button → validates all required fields; shows field-level errors on failure
  - On success, navigates to request detail page

## Acceptance Checks
- [ ] Request list shows requests filtered by status tab
- [ ] Table is sortable by column header
- [ ] New request form saves as draft without validation
- [ ] Submit validates required fields and shows errors
- [ ] Edit form pre-fills existing request data
- [ ] Navigation from list → form → detail works correctly
