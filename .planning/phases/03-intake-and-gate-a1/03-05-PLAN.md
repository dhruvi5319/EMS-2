# Plan 03-05: Gate A1 Decision Panel + Review Queue UI

**Phase:** 3 — Intake and Gate A1  
**Status:** not started  
**Features:** F3  

## Goal

Build the Gate A1 Decision Panel in the request detail page and the Review Queue page that lists pending gate actions for the current user.

## Tasks

### Task 1 — Gate A1 Decision Panel
- Create `src/components/gates/GateA1Panel.tsx`:
  - Shown when: request.status = submitted AND user has AL or AD role
  - RadioGroup: risk level (Low / Medium / High)
  - Textarea: rationale (required)
  - "Approve" button → triggers AlertDialog confirmation → calls `POST /api/requests/:id/gate/a1/approve`
  - "Decline" button → triggers AlertDialog confirmation → calls `POST /api/requests/:id/gate/a1/decline`
  - On success: in-place React state update (no window.location.reload) shows approved/declined banner
  - AlertDialog uses custom async buttons (not AlertDialogAction) to support loading spinner
- Create `src/components/gates/GateA1DecidedCard.tsx`:
  - Shows when request is accepted or declined
  - Fetches `GET /api/requests/:id/gate/decision`
  - Displays: decision (approved/declined), approver display_name, rationale, date

### Task 2 — Review Queue page
- Create `src/hooks/useReviewQueue.ts`: fetches submitted requests (for AL/AD)
- Create `src/pages/ReviewQueuePage.tsx`:
  - Table of submitted requests pending A1 decision
  - Columns: request type, topic, requester, submitted date, due date
  - Row click navigates to request detail

### Task 3 — Wire gate panel into request detail
- In `RequestDetailPage.tsx`, conditionally render `<GateA1Panel>` or `<GateA1DecidedCard>` in the `data-section="gate-a1"` slot based on request status and user roles

## Acceptance Checks
- [ ] AL user sees Gate A1 Panel on submitted request detail
- [ ] Risk level is required before approving
- [ ] Rationale is required before approving or declining
- [ ] Approval: banner shows "Request Approved", engagement is created
- [ ] Decline: banner shows "Request Declined", no engagement
- [ ] GateA1DecidedCard shows correct approver name and rationale
- [ ] Review Queue lists all submitted requests
