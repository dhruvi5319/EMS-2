# Plan 03-02: Gate A1 Backend

**Phase:** 3 — Intake and Gate A1  
**Status:** not started  
**Features:** F3  

## Goal

Implement the atomic Gate A1 decision backend: approve (auto-creates engagement shell with ENG-YYYY-NNNNN job code) and decline (records rationale, no engagement created). All decisions write immutable GateDecision and AuditEvent records.

## Tasks

### Task 1 — Gate A1 service
- Create `src/modules/gates/gates.repository.ts`: createGateDecision, getGateDecisions, getLatestGateDecision
- Create `src/modules/gates/gates.service.ts`:
  - `approveA1(requestId, actorId, { risk_level, rationale })`:
    - Validates request is in `submitted` status
    - Generates job code: `ENG-{YYYY}-{COUNT+1:05d}` using `COUNT(engagements)+1` in transaction
    - Creates Engagement record (phase=planning, status=active)
    - Sets request.status = accepted
    - Creates GateDecision (gate_type=A1, status=approved)
    - Creates AuditEvent (action=GATE_A1_APPROVED)
    - All in a single DB transaction
  - `declineA1(requestId, actorId, { rationale })`:
    - Validates request is in `submitted` status
    - Sets request.status = declined
    - Creates GateDecision (gate_type=A1, status=declined, engagement_id — note: schema requires NOT NULL so use a synthetic approach or store on request)
    - Creates AuditEvent (action=GATE_A1_DECLINED)

### Task 2 — Gate A1 routes
- Create `src/modules/gates/gates.controller.ts`:
  - `POST /api/requests/:id/gate/a1/approve` — (AL, AD) — approve with risk_level and rationale
  - `POST /api/requests/:id/gate/a1/decline` — (AL, AD) — decline with rationale
  - `GET /api/requests/:id/gate/decision` — get latest A1 decision for display

## Acceptance Checks
- [ ] `POST /api/requests/:id/gate/a1/approve` creates engagement with correct ENG-YYYY-NNNNN job code
- [ ] Approval sets request.status = accepted and engagement.phase = planning
- [ ] GateDecision and AuditEvent rows are written on approval
- [ ] `POST /api/requests/:id/gate/a1/decline` sets request.status = declined; no engagement created
- [ ] Both operations are atomic (transaction rollback on failure)
- [ ] Attempting to approve an already-decided request returns 409
