# Plan 03-01: Request Backend API

**Phase:** 3 — Intake and Gate A1  
**Status:** not started  
**Features:** F2  

## Goal

Build the complete request lifecycle backend API: CRUD, file upload with multer, StorageProvider integration, and file replace behavior.

## Tasks

### Task 1 — Storage provider
- Create `src/storage/storage.interface.ts` (IStorageProvider interface: put, get, delete, exists)
- Create `src/storage/local.storage.ts` (filesystem implementation using STORAGE_BASE_PATH)
- Create `src/storage/index.ts` (factory: returns local or s3 provider based on STORAGE_BACKEND env var)

### Task 2 — Request CRUD API
- Create `src/modules/requests/requests.repository.ts`: listRequests, getRequest, createRequest, updateRequest
- Create `src/modules/requests/requests.service.ts`:
  - `listRequests(userId, userRoles)` — AL/AD see all; EM sees accepted requests
  - `getRequest(id)` — returns request with intake document metadata
  - `createRequest(data, userId)` — creates draft request
  - `updateRequest(id, data)` — updates draft request fields
  - `submitRequest(id)` — validates required fields; sets status=submitted; writes AuditEvent
  - Validation: request_type, requester, topic, agency_program, due_date all required before submit
- Create `src/modules/requests/requests.controller.ts`:
  - `GET /api/requests` — list (AL, EM, AD)
  - `POST /api/requests` — create draft (AL, AD)
  - `GET /api/requests/:id` — get detail (AL, EM, AD)
  - `PATCH /api/requests/:id` — update draft (AL, AD)
  - `POST /api/requests/:id/submit` — submit (AL, AD)

### Task 3 — Intake file upload
- Add multer middleware (memory storage; max 25 MB; MIME type whitelist: pdf, docx, doc, xlsx, xls, csv, txt, png, jpg)
- `POST /api/requests/:id/upload` — upload intake document; replaces previous file via StorageProvider.delete + put; updates intake_document_ref and intake_document_name; writes AuditEvent
- `GET /api/requests/:id/file` — stream intake document; access-controlled

## Acceptance Checks
- [ ] `POST /api/requests` creates a draft request
- [ ] `POST /api/requests/:id/submit` validates required fields and returns 422 with field errors if missing
- [ ] `POST /api/requests/:id/upload` uploads file and replaces previous file on re-upload
- [ ] `GET /api/requests/:id/file` streams the intake document
- [ ] File size >25 MB returns 413
