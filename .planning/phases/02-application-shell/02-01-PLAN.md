# Plan 02-01: shadcn/ui Initialization + Token Migration

**Phase:** 2 — Application Shell  
**Status:** not started  
**Features:** F0  

## Goal

Initialize the shadcn/ui component system with the new-york style and CSS variable token configuration. Migrate Phase 1 components to use the token system. All future UI components will use shadcn primitives.

## Tasks

### Task 1 — shadcn/ui initialization
- Add shadcn/ui configuration to `components.json` (new-york style, CSS variables enabled)
- Configure CSS variable tokens in `src/index.css` (`:root` block with --primary, --secondary, --background, --foreground, --border, --ring, --destructive, etc.)
- Extend `tailwind.config.ts` to consume CSS variables via `hsl(var(--...))` pattern
- Manually create core shadcn components (registry is unavailable; write from new-york templates):
  - `src/components/ui/button.tsx`
  - `src/components/ui/input.tsx`
  - `src/components/ui/label.tsx`
  - `src/components/ui/card.tsx`
  - `src/components/ui/badge.tsx`
  - `src/components/ui/dialog.tsx`
  - `src/components/ui/alert.tsx`

### Task 2 — Token migration
- Update `LoginPage.tsx` to use shadcn Button, Input, Label, Card components
- Update `AppShell.tsx` sidebar and topbar to use token-based colors (bg-background, text-foreground, border-border)
- Verify visual consistency: login form and shell look polished with the new token system

## Acceptance Checks
- [ ] CSS variable tokens defined in `:root` in `index.css`
- [ ] Tailwind theme extension uses `hsl(var(--primary))` pattern
- [ ] Login page uses shadcn Button, Input, Label, Card
- [ ] App shell sidebar uses token-based colors
- [ ] No visual regressions on login or dashboard pages
