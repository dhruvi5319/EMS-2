# Plan 02-05: User Management Page

**Phase:** 2 — Application Shell  
**Status:** not started  
**Features:** F0  

## Goal

Build the Admin-only user management page where admins can create users, assign roles, and deactivate/activate accounts.

## Tasks

### Task 1 — shadcn components for user management
- Manually create `src/components/ui/table.tsx` from new-york template
- Manually create `src/components/ui/select.tsx` from new-york template
- Manually create `src/components/ui/checkbox.tsx` from new-york template
- Manually create `src/components/ui/alert-dialog.tsx` from new-york template

### Task 2 — User management page components
- Create `src/pages/AdminPage.tsx` (Admin-only, RoleGuard wraps page)
- Create `src/components/admin/UserTable.tsx`: list all users with columns: name, email, roles, status, actions
- Create `src/components/admin/CreateUserDialog.tsx`: Dialog with form (email, display_name, password, roles multi-select); calls `POST /api/users`
- Create `src/components/admin/EditUserRolesDialog.tsx`: Dialog to update roles; calls `PUT /api/users/:id/roles`
- Create `src/components/admin/DeactivateUserConfirm.tsx`: AlertDialog to confirm deactivation; calls `PUT /api/users/:id/deactivate`

### Task 3 — Playwright E2E tests
- Create `e2e/admin.spec.ts`:
  - Test: non-AD user redirected to 403 at /admin
  - Test: AD user sees user list
  - Test: create user → appears in list
  - Test: edit roles → updated in list
  - Test: deactivate user → status changes

## Acceptance Checks
- [ ] Non-AD user sees 403 when accessing /admin
- [ ] AD user can view, create, edit roles, and deactivate/activate users
- [ ] Created user can log in with the provided password
- [ ] Role changes take effect on next request
- [ ] Deactivated user's existing sessions are invalidated
