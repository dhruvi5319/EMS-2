# Plan 02-03: Backend APIs — Search, User Management, Audit Trail

**Phase:** 2 — Application Shell  
**Status:** not started  
**Features:** F0  

## Goal

Implement the three backend APIs needed for the application shell: global search, user management (Admin), and audit trail per engagement.

## Tasks

### Task 1 — Global search API
- Create `src/modules/search/search.service.ts`:
  - `GET /api/search?q=<query>` — searches engagements by job_code, title, portfolio
  - Scoping: non-AD users see only engagements where they have a team_assignment; AD sees all
  - Returns array of `{ id, job_code, title, phase, status, owner_name }`
  - Min query length: 2 characters; max results: 20

### Task 2 — User management API (Admin only)
- Create `src/modules/users/users.repository.ts`, `users.service.ts`, `users.controller.ts`:
  - `GET /api/users` — list all users with roles (AD only)
  - `POST /api/users` — create user with roles (AD only); hash password with bcryptjs
  - `PUT /api/users/:id/roles` — replace user roles (AD only)
  - `PUT /api/users/:id/deactivate` — deactivate user; invalidates all sessions (AD only)
  - `PUT /api/users/:id/activate` — reactivate user (AD only)

### Task 3 — Audit trail API
- Create `src/modules/audit/audit.repository.ts`, `audit.service.ts`, `audit.controller.ts`:
  - `GET /api/engagements/:id/audit` — list audit events for engagement, paginated, filterable by action and date range
  - Returns: id, actor_name, action, object_type, summary, occurred_at
  - Accessible to all authenticated users with access to the engagement

## Acceptance Checks
- [ ] `GET /api/search?q=eng` returns matching engagements scoped to user's access
- [ ] Non-AD user cannot see engagements they are not assigned to in search results
- [ ] `POST /api/users` creates user with hashed password and assigned roles (AD only)
- [ ] `PUT /api/users/:id/roles` updates roles; non-AD returns 403
- [ ] `PUT /api/users/:id/deactivate` revokes all sessions for the deactivated user
- [ ] `GET /api/engagements/:id/audit` returns paginated audit events
