# Plan 02-06: Audit Trail View

**Phase:** 2 — Application Shell  
**Status:** not started  
**Features:** F0  

## Goal

Build the audit trail view for each engagement — showing a chronological, filterable timeline of audit events.

## Tasks

### Task 1 — shadcn components for audit trail
- Manually create `src/components/ui/scroll-area.tsx` from new-york template

### Task 2 — Audit trail page components
- Create `src/hooks/useAuditTrail.ts`: fetches `GET /api/engagements/:id/audit` with pagination and filters
- Create `src/components/audit/ActionCodeBadge.tsx`: color-coded badge for action type (GATE_APPROVED, EVIDENCE_UPLOADED, etc.)
- Create `src/components/audit/AuditEventCard.tsx`: single audit event card (actor, action badge, summary, timestamp)
- Create `src/components/audit/AuditTrailFilters.tsx`: filter form (action type select, date range inputs); committed on Apply button
- Create `src/components/audit/AuditTimeline.tsx`: vertical timeline of AuditEventCard items
- Create `src/pages/AuditTrailPage.tsx`: combines filters + timeline + pagination controls
- Wire `/engagements/:id/audit` route in App.tsx

### Task 3 — Playwright E2E tests
- Create `e2e/audit.spec.ts`:
  - Test: audit trail page loads for an engagement
  - Test: filter by action type shows only matching events
  - Test: date range filter works correctly
  - Test: pagination loads next page

## Acceptance Checks
- [ ] Audit trail page loads at `/engagements/:id/audit`
- [ ] Events display in reverse chronological order with actor, action, summary, timestamp
- [ ] Filter by action type filters the list correctly
- [ ] Date range filter works
- [ ] Pagination works (next/prev page)
- [ ] Accessible to all authenticated users with engagement access
