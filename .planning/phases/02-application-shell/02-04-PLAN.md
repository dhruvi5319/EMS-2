# Plan 02-04: Global Search UI

**Phase:** 2 — Application Shell  
**Status:** not started  
**Features:** F0  

## Goal

Build the global search UI using the shadcn Command component as a keyboard-accessible overlay triggered by a search bar in the TopBar and the ⌘K / Ctrl+K shortcut.

## Tasks

### Task 1 — shadcn Command component
- Manually create `src/components/ui/command.tsx` from the new-york template (wraps cmdk)
- Manually create `src/components/ui/popover.tsx` from the new-york template

### Task 2 — Global search bar and overlay
- Create `src/hooks/useSearch.ts`: debounced search hook (300ms), calls `GET /api/search?q=`
- Create `src/components/GlobalSearchBar.tsx`:
  - Input in TopBar that opens Command overlay on focus
  - ⌘K / Ctrl+K keyboard shortcut opens overlay
  - Overlay closes on Escape or outside click
  - Shows results grouped by engagement; keyboard navigation (arrow keys, Enter)
  - Selecting a result navigates to `/engagements/:id`
  - Overlay only opens when query.length > 0
- Mount `GlobalSearchBar` in `src/layout/AppShell.tsx` TopBar (flex-1, mx-6, max-w-[640px])

### Task 3 — Playwright E2E tests
- Create `e2e/search.spec.ts`:
  - Test: click search bar → overlay opens
  - Test: type query → results appear
  - Test: Ctrl+K → overlay opens
  - Test: Escape → overlay closes
  - Test: select result → navigate to engagement

## Acceptance Checks
- [ ] Search bar is visible and clickable in the TopBar
- [ ] Typing in the search bar shows matching engagements
- [ ] ⌘K / Ctrl+K opens the search overlay
- [ ] Escape closes the overlay
- [ ] Selecting a result navigates to the correct engagement
- [ ] Empty query does not open overlay
