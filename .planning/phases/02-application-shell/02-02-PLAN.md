# Plan 02-02: Role-Filtered Navigation Sidebar

**Phase:** 2 — Application Shell  
**Status:** not started  
**Features:** F0  

## Goal

Implement role-filtered navigation so each role only sees sections they are authorized to access. Direct URL access to a hidden section returns a 403 Forbidden page.

## Tasks

### Task 1 — Navigation configuration and RoleGuard
- Define `NAV_SECTIONS` config in `src/layout/nav-config.ts`:
  - Dashboard (all roles)
  - Requests (AL, EM, AD)
  - Engagements (AL, EM, AN, QA, IR, PC, AD)
  - Evidence (AN, EM, QA, IR, AD)
  - Review Queue (AL, EM, QA, PC, AD)
  - Reports/Dashboard (EM, QA, AD, AL, PC, RO)
  - User Management (AD only)
- Create `src/components/RoleGuard.tsx`:
  - Checks `user.roles` against required roles
  - Renders `<ForbiddenPage />` in-place (URL stays, shell remains visible) if unauthorized

### Task 2 — Role-filtered sidebar and forbidden page
- Update `src/layout/AppShell.tsx` NavigationRail to use `NAV_SECTIONS` config
- Filter nav links based on `user.roles` (empty allowedRoles = all roles permitted)
- Create `src/pages/ForbiddenPage.tsx`: 403 message with link back to dashboard
- Wrap each page route with `<RoleGuard>` in `App.tsx`
- Create `src/components/ReviewQueueBadge.tsx`: shows count of pending gate actions (stub count for now)

## Acceptance Checks
- [ ] AN user does not see Requests in nav; direct URL to /requests returns 403 page
- [ ] RO user does not see Review Queue; direct URL to /review-queue returns 403 page
- [ ] AD user sees all nav sections
- [ ] Shell sidebar and topbar remain visible on 403 page
- [ ] Empty allowedRoles means all roles can access
